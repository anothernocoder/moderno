**Contact** es un [block](/es/introduction/blocks/) de marketing: una sección de contacto con texto
introductorio y una lista de datos de contacto a la izquierda, y un formulario de
nombre/correo/asunto/mensaje a la derecha. Compone los primitives `Input`, `Textarea` y `Button` y
los tokens `--md-*`. Lo copias a tu repo con el CLI y lo editas libremente.

## Props

| Prop | Tipo | Default | Descripción |
|------|------|---------|-------------|
| `eyebrow` | `string` | `'Contacto'` | Etiqueta pequeña sobre el título |
| `title` | `string` | `'¿Hablamos?'` | Título de la sección |
| `description` | `string` | `'Escríbenos y te respondemos en menos de 24 horas.'` | Texto de introducción corto |
| `contactInfo` | `ContactInfoItem[]` | Ejemplos de email/teléfono/oficina | Lista de datos de contacto |
| `nameLabel` | `string` | `'Nombre'` | Label del campo de nombre |
| `emailLabel` | `string` | `'Correo'` | Label del campo de correo |
| `subjectLabel` | `string` | `'Asunto'` | Label del campo de asunto |
| `messageLabel` | `string` | `'Mensaje'` | Label del campo de mensaje |
| `submitLabel` | `string` | `'Enviar mensaje'` | Label del botón de envío |
| `defaultValues` | `Partial<ContactFormValues>` | `{}` | Valores iniciales de los campos del formulario |
| `onSubmit` | `(values: ContactFormValues) => void` | — | Se llama con los valores del formulario al enviarlo (`preventDefault` ya está manejado) |

Un `ContactInfoItem` es `{ label: string; value: string; href?: string }`. `ContactFormValues` es
`{ name: string; email: string; subject: string; message: string }`.

## Uso

```tsx
import { Contact } from './blocks/Contact'

<Contact
  eyebrow="Contacto"
  title="¿Hablamos?"
  description="Escríbenos y te respondemos en menos de un día."
  contactInfo={[
    { label: 'Email', value: 'hola@acme.com', href: 'mailto:hola@acme.com' },
    { label: 'Teléfono', value: '+57 300 010 1234', href: 'tel:+573000101234' },
  ]}
  nameLabel="Nombre"
  emailLabel="Correo"
  subjectLabel="Asunto"
  messageLabel="Mensaje"
  submitLabel="Enviar mensaje"
  onSubmit={(values) => console.log(values)}
/>
```

La API de props es idéntica en los cuatro frameworks (en Vue, `onSubmit` es el evento `@submit` en
lugar de un prop).

El block renderiza una columna de introducción (eyebrow, título, descripción, datos de contacto)
junto a un formulario con nombre y correo lado a lado, un campo de asunto, un textarea de mensaje y
un botón de envío. Maneja internamente el estado de sus campos y llama a `onSubmit` con los valores
actuales; conecta ese callback a tu propia lógica de request.
