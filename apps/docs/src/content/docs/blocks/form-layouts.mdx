**Form Layouts** is an application [block](/introduction/blocks/): the page-level layout
scaffolding for a settings/profile-edit form. Two grouped sections ("Profile" and
"Preferences"), each with a heading, a description, and a responsive multi-column field
grid (first/last name side by side, full-width fields below), separated by a `Divider` and
followed by a footer action row. It composes the `Input`, `Textarea`, `Select`, `Divider`,
and `Button` primitives and the `--md-*` tokens. You copy it into your repo with the CLI and
edit it freely.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `defaultValues` | `Partial<FormLayoutsValues>` | sample profile | Initial field values |
| `onSave` | `(values: FormLayoutsValues) => void` | â€” | Called on submit with the current field values |
| `onCancel` | `() => void` | â€” | Called when "Cancel" is clicked |

`FormLayoutsValues` shape:

| Field | Type | Description |
|-------|------|--------------|
| `firstName` | `string` | First name |
| `lastName` | `string` | Last name |
| `email` | `string` | Email address |
| `bio` | `string` | Free-text bio, shown in the "Profile" section |
| `role` | `string` | Selected role option value, from the "Preferences" section |
| `timezone` | `string` | Selected timezone option value, from the "Preferences" section |

## Usage

```tsx
import { FormLayouts } from './blocks/FormLayouts'

<FormLayouts
  defaultValues={{ firstName: 'Ada', lastName: 'Lovelace', email: 'ada@acme.com' }}
  onSave={(values) => saveProfile(values)}
  onCancel={() => navigateBack()}
/>
```

The props API is identical across all four frameworks. In Vue, `onSave`/`onCancel` are the
`@save`/`@cancel` events; in the others they're callback props.
