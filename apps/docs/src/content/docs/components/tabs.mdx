**Tabs** lets you switch between sibling views within the same context. It is a
**compositional** primitive (ADR-0003): it exposes parts that share a single Zag state
machine — `Tabs.Root`, `Tabs.List`, `Tabs.Trigger`, `Tabs.Content`. The behavior (arrow-key
navigation, `Home`/`End`, focus, ARIA `role="tab"`/`tabpanel`) comes from Zag's `tabs` machine.

## Live demo

The same component, rendered from each framework. Switch tabs with the mouse or with
the keyboard arrow keys.

## Usage

## Anatomy and props

| Part | Prop | Description |
|-------|------|-------------|
| `Tabs.Root` | `defaultValue` | Initially selected tab (uncontrolled). |
| `Tabs.Root` | `value` | Selected tab (controlled). |
| `Tabs.Root` | `onValueChange` | Callback `(value: string) => void` when the tab changes. |
| `Tabs.Trigger` | `value`, `disabled` | Identifies the tab; `disabled` disables it. |
| `Tabs.Content` | `value` | Associates the panel with the tab that has the same `value`. |

In Vue, the controlled `value` is bound with `v-model`; in Svelte, with `bind:value`.

## Accessibility

- Full ARIA (`role="tablist"`/`tab"`/`tabpanel"`, `aria-selected`, `aria-controls`)
  provided by Zag's `tabs` machine.
- Keyboard navigation: arrow keys to move, `Home`/`End` to jump to the ends.
- The active panel is focusable; the focus indicator uses `--md-primary`.
