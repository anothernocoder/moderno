**Order Summaries** is an ecommerce [block](/introduction/blocks/): a read-only order
summary panel listing line items — each with a title, quantity, and line total —
followed by subtotal, shipping, and tax rows, and a final total. It's typically shown
alongside a checkout form or an order confirmation. It composes the `Divider` primitive
plus the `--md-*` tokens. You copy it into your repo with the CLI and edit it freely.

This block is purely presentational: unlike the [Shopping Cart](/blocks/shopping-cart/)
block, there are no quantity steppers or remove actions — quantities and totals are
computed once from the `items` prop and rendered as static text.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `heading` | `string` | `'Resumen del pedido'` | Heading above the panel (omit to hide it) |
| `items` | `OrderSummariesItem[]` | 3 sample line items | Order line items — each `{ id, title, price, quantity? }` |
| `emptyMessage` | `string` | `'No hay productos en el pedido.'` | Message shown when `items` is empty |
| `subtotalLabel` | `string` | `'Subtotal'` | Label for the subtotal row |
| `shippingLabel` | `string` | `'Envío'` | Label for the shipping row |
| `shipping` | `number` | `15000` | Shipping cost added to the total; `0` renders `freeShippingLabel` instead |
| `freeShippingLabel` | `string` | `'Gratis'` | Text shown in the shipping row when `shipping` is `0` |
| `taxLabel` | `string` | `'Impuestos'` | Label for the tax row |
| `taxRate` | `number` | `0.19` | Tax rate applied to the subtotal to compute the tax row (e.g. `0.19` = 19%) |
| `totalLabel` | `string` | `'Total'` | Label for the final total row |

## Usage

```tsx
import { OrderSummaries } from './blocks/OrderSummaries'

<OrderSummaries
  items={[
    { id: 'chair', title: 'Oak Dining Chair', price: 389000, quantity: 1 },
    { id: 'lamp', title: 'Arc Floor Lamp', price: 128000, quantity: 2 },
  ]}
  shipping={0}
/>
```

The props API is identical across all four frameworks. Each line item's `price` is a
plain `number` (formatted internally with `toLocaleString`); the subtotal is the sum of
`price * quantity` across all items, computed once from the `items` prop — there's no
local editable state, so passing new `items` re-renders the totals accordingly. Tax is
derived from the subtotal via `taxRate`, so you can pass a fixed `taxRate` of `0` and
supply your own pre-computed tax through a custom `taxLabel`/row if your backend already
returns a tax amount rather than a rate. No new primitives, no interactive
controls — just the existing `Divider` primitive composed with layout, unlike the
[Shopping Cart](/blocks/shopping-cart/) block's quantity editing.
