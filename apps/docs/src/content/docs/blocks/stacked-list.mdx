**Stacked List** is an application [block](/introduction/blocks/): a vertically stacked list
of records where each row is its own card, with more visual separation and weight than a
plain list row. Each card shows an avatar, primary/secondary text, an optional status badge,
trailing meta, and a row action. It composes the `Card`, `Avatar`, `Badge`, and `Button`
primitives and the `--md-*` tokens. You copy it into your repo with the CLI and edit it freely.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `items` | `StackedListItem[]` | 3 sample tickets | The records to render |
| `onItemAction` | `(item: StackedListItem) => void` | — | Called when a record's `⋯` action button is clicked |

`StackedListItem` shape:

| Field | Type | Description |
|-------|------|--------------|
| `id` | `string` | Unique record id |
| `title` | `string` | Primary text |
| `subtitle` | `string` | Secondary text, shown under the title |
| `avatarInitials` | `string` | Fallback initials for the `Avatar` |
| `avatarSrc` | `string` | Image URL for the `Avatar` |
| `status` | `string` | Status label shown in the `Badge` (omit to hide the badge) |
| `statusVariant` | `'neutral' \| 'solid' \| 'success' \| 'warning' \| 'error' \| 'info'` | `Badge` color variant for the status |
| `meta` | `string` | Trailing meta text, e.g. a relative timestamp |

## Usage

```tsx
import { StackedList } from './blocks/StackedList'

<StackedList
  items={[
    { id: 'T-1', title: 'Cannot export report', subtitle: 'Jane Doe · Support', avatarInitials: 'JD', status: 'Open', statusVariant: 'warning', meta: '12m ago' },
    { id: 'T-2', title: 'Requests plan upgrade', subtitle: 'John Roe · Billing', avatarInitials: 'JR', status: 'Resolved', statusVariant: 'success', meta: '1h ago' },
  ]}
  onItemAction={(item) => openRecordMenu(item)}
/>
```

The props API is identical across all four frameworks. In Vue, `onItemAction` is the
`@item-action` event; in the others it's a callback prop.
