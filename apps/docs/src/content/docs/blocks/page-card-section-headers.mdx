**Page/Card/Section headers** is an application [block](/introduction/blocks/): a family of
three related header patterns for application views — a **page header** (breadcrumbs, title,
description, actions, and a status/meta row), a **card header** (title, subtitle, and actions
for a card/panel), and a **section** with a top header bar (title, description, actions) and a
bottom bar (summary and actions). It composes the `Button`, `Badge`, `Card`, and `Divider`
primitives and the `--md-*` tokens. You copy it into your repo with the CLI and edit it freely.

the page header, card header, or section bars into their own components if you need to reuse
them independently.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `breadcrumbs` | `{ label: string; href?: string }[]` | 3 sample crumbs | Breadcrumb trail above the page title |
| `pageTitle` | `string` | `'Rediseño Q3'` | Page header title |
| `pageDescription` | `string` | sample copy | Page header description, under the title |
| `pageStatus` | `string` | `'Activo'` | Status label shown as a dot `Badge` in the meta row |
| `pageMeta` | `string` | sample copy | Meta text next to the status badge |
| `primaryLabel` | `string` | `'Nueva tarea'` | Page header primary action button label (hidden if empty) |
| `onPrimaryAction` | `() => void` | — | Called when the primary action button is clicked |
| `secondaryLabel` | `string` | `'Compartir'` | Page header secondary action button label (hidden if empty) |
| `onSecondaryAction` | `() => void` | — | Called when the secondary action button is clicked |
| `cardTitle` | `string` | `'Miembros del equipo'` | Card header title |
| `cardSubtitle` | `string` | sample copy | Card header subtitle |
| `cardBody` | `string` | sample copy | Card body text, shown below a `Divider` (hidden if empty) |
| `cardActionLabel` | `string` | `'Invitar'` | Card header action button label (hidden if empty) |
| `onCardAction` | `() => void` | — | Called when the card header action button is clicked |
| `sectionTitle` | `string` | `'Actividad reciente'` | Section top bar title |
| `sectionDescription` | `string` | sample copy | Section top bar description |
| `sectionActionLabel` | `string` | `'Ver todo'` | Section top bar action button label (hidden if empty) |
| `onSectionAction` | `() => void` | — | Called when the section top bar action button is clicked |
| `sectionBody` | `string` | sample copy | Section body content between the top and bottom bars |
| `sectionSummary` | `string` | sample copy | Section bottom bar summary text |
| `sectionBarActionLabel` | `string` | `'Exportar'` | Section bottom bar secondary action label (hidden if empty) |
| `onSectionBarAction` | `() => void` | — | Called when the section bottom bar secondary action is clicked |
| `sectionBarPrimaryLabel` | `string` | `'Marcar completadas'` | Section bottom bar primary action label (hidden if empty) |
| `onSectionBarPrimaryAction` | `() => void` | — | Called when the section bottom bar primary action is clicked |

## Usage

```tsx
import { PageCardSectionHeaders } from './blocks/PageCardSectionHeaders'

<PageCardSectionHeaders
  pageTitle="Order #A-102"
  pageDescription="Track fulfillment and manage this order."
  primaryLabel="Refund"
  onPrimaryAction={() => refundOrder()}
  cardTitle="Customer"
  cardActionLabel="Edit"
  onCardAction={() => editCustomer()}
  sectionTitle="Timeline"
  sectionSummary="4 events"
/>
```

The props API is identical across all four frameworks. In Vue, `onPrimaryAction`/
`onSecondaryAction`/`onCardAction`/`onSectionAction`/`onSectionBarAction`/
`onSectionBarPrimaryAction` are the `@primary-action`/`@secondary-action`/`@card-action`/
`@section-action`/`@section-bar-action`/`@section-bar-primary-action` events; in the others
they're callback props.
