**EmptyStates** es un [block](/es/introduction/blocks/) de aplicación: el patrón clásico "aún no
hay datos" que se muestra en lugar de una tabla, lista o sección vacía — un icono en una caja con
borde, un título, una descripción breve y un CTA principal para "crear tu primer X". No hay
asset de ilustración — el icono es una caja con borde como los iconos de item de Features,
consistente con cómo los blocks de Moderno evitan assets de imagen binarios. Compone el primitive
`Button` y los tokens `--md-*`. Lo copias a tu repo con el CLI y lo editas libremente.

## Props

| Prop | Tipo | Default | Descripción |
|------|------|---------|-------------|
| `icon` | `string` | `'📭'` | Emoji/glifo mostrado dentro de la caja con borde |
| `title` | `string` | `'Aún no tienes elementos'` | Título debajo del icono |
| `description` | `string` | `'Cuando crees tu primer elemento, aparecerá aquí.'` | Texto de apoyo |
| `actionLabel` | `string` | `'Crear el primero'` | Etiqueta del `Button` principal |
| `onAction` | `() => void` | — | Se llama al hacer clic en el CTA principal |
| `secondaryLabel` | `string` | `undefined` | Etiqueta opcional para un link secundario |
| `secondaryHref` | `string` | `undefined` | `href` opcional para el link secundario |

`secondaryLabel`/`secondaryHref` se renderizan como un `<a>` subrayado junto al botón
principal — útil para un enlace de escape tipo "saber más" o "importar en su lugar". Solo se
renderiza cuando ambos están presentes.

## Uso

```tsx
import { EmptyStates } from './blocks/EmptyStates'

// Default: aún no se ha creado nada
<EmptyStates
  title="Aún no hay proyectos"
  description="Crea tu primer proyecto para empezar."
  actionLabel="Crear proyecto"
  onAction={() => openCreateProjectModal()}
/>

// Variante: sin resultados de búsqueda/filtro
<EmptyStates
  icon="🔍"
  title="Sin resultados"
  description="Intenta ajustar tu búsqueda o filtros."
  actionLabel="Limpiar búsqueda"
  onAction={() => clearSearch()}
/>
```

La API de props es idéntica en los cuatro frameworks. En Vue y Svelte, `onAction` es un evento
`@action` / una prop callback `onAction` respectivamente, igual que los demás blocks de
aplicación (como [Table](/es/blocks/table/)) exponen sus callbacks.
