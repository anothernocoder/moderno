**List** is an application [block](/introduction/blocks/): a companion to [Table](/blocks/table/)
for contexts where a simple row-based list reads better than a data grid. Each row shows an
avatar, primary/secondary text, an optional status badge, optional meta text, and a row action —
with no sorting, no multi-select, and no tabular columns. It composes the `Avatar`, `Badge`,
`Button`, and `Divider` primitives and the `--md-*` tokens.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `items` | `ListItem[]` | 4 sample records | The rows to render |
| `onItemAction` | `(item: ListItem) => void` | — | Called when a row's `⋯` action button is clicked |

`ListItem` shape:

| Field | Type | Description |
|-------|------|--------------|
| `id` | `string` | Unique row id |
| `title` | `string` | Primary text |
| `subtitle` | `string` | Secondary text (omitted if not set) |
| `meta` | `string` | Trailing meta text, e.g. a timestamp (hidden if omitted) |
| `avatarSrc` | `string` | Avatar image URL |
| `avatarInitials` | `string` | Avatar fallback initials, shown when `avatarSrc` is absent |
| `status` | `string` | Status label shown in the `Badge` (hidden if omitted) |
| `statusVariant` | `'neutral' \| 'solid' \| 'success' \| 'warning' \| 'error' \| 'info'` | `Badge` color variant for the status |

## Usage

```tsx
import { List } from './blocks/List'

<List
  items={[
    { id: '1', title: 'Acme Co', subtitle: 'billing@acme.co', avatarInitials: 'AC', status: 'Active', statusVariant: 'success', meta: '2h ago' },
    { id: '2', title: 'Globex', subtitle: 'ops@globex.com', avatarInitials: 'GX', status: 'Pending', statusVariant: 'warning', meta: '1d ago' },
  ]}
  onItemAction={(item) => openItemMenu(item)}
/>
```

The props API is identical across all four frameworks. In Vue, `onItemAction` is the
`@item-action` event; in the others it's a callback prop.
