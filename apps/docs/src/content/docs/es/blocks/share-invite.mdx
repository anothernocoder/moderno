**ShareInvite** es un [block](/es/introduction/blocks/) de marketing (structure doc §2.3 "Data
display" — "Share and Invite"): un enlace compartible con acción de copiar, una fila de botones de
canal/red social y un formulario de invitación por correo con una lista de invitaciones
pendientes. Es puramente presentacional — no hay acceso al portapapeles ni lógica async adentro,
solo callbacks (`onCopyLink`, `onShare`, `onInvite`) que el consumidor conecta a su propia escritura
al portapapeles, integración de share y request de invitación. Compone los primitives
`Input`/`Button`/`Badge` y los tokens `--md-*`. Lo copias a tu repo con el CLI y lo editas
libremente.

## Props

| Prop | Tipo | Default | Descripción |
|------|------|---------|-------------|
| `title` | `string` | `'Comparte e invita'` | Título de la sección |
| `description` | `string` | Una oración corta | Copy de apoyo opcional |
| `shareUrl` | `string` | Un enlace de ejemplo | El enlace compartible, renderizado como solo lectura en el campo de copia |
| `linkLabel` | `string` | `'Tu enlace para compartir'` | Label sobre el campo del enlace |
| `copyLabel` | `string` | `'Copiar'` | Label del botón de copiar |
| `onCopyLink` | `(url: string) => void` | — | Se dispara al hacer click en copiar — acá va tu escritura real al portapapeles |
| `channels` | `ShareChannel[]` | Canales de ejemplo Correo/X/WhatsApp | Botones de canal renderizados junto al enlace |
| `onShare` | `(channel: ShareChannel) => void` | — | Se dispara al hacer click en un botón de canal |
| `inviteEmail` | `string` | `''` | Valor controlado del campo de invitación por correo |
| `onInviteEmailChange` | `(value: string) => void` | — | Se dispara al cambiar el campo de correo |
| `inviteEmailPlaceholder` | `string` | `'nombre@empresa.com'` | Placeholder del campo de correo |
| `inviteLabel` | `string` | `'Invitar'` | Label del botón de invitar |
| `onInvite` | `(email: string) => void` | — | Se dispara al enviar el formulario de invitación |
| `invites` | `ShareInviteItem[]` | `undefined` | Invitaciones pendientes/aceptadas renderizadas bajo el formulario; la sección se oculta si está vacía |
| `invitesLabel` | `string` | `'Invitaciones pendientes'` | Encabezado sobre la lista de invitaciones |

## Uso

```tsx
import { ShareInvite } from './blocks/ShareInvite'

<ShareInvite
  shareUrl="https://moderno.app/invite/abc123"
  onCopyLink={(url) => navigator.clipboard.writeText(url)}
  onShare={(channel) => console.log('share via', channel.id)}
  inviteEmail={email}
  onInviteEmailChange={setEmail}
  onInvite={(value) => addInvite(value)}
  invites={invites}
/>
```

La API de props es idéntica en los cuatro frameworks.

El block renderiza un panel con borde sobre `var(--md-surface-raised)` — el campo de enlace y el
botón de copiar, la fila de botones de canal, el formulario de invitación por correo y (solo
cuando `invites` no está vacío) una lista con borde de filas de invitación con un correo y un
`Badge` de estado. Este block no tiene lógica async interna: `onCopyLink` se dispara con el
`shareUrl` actual para que el consumidor haga la escritura real al portapapeles, `onShare` se
dispara con el canal clickeado para que el consumidor haga la integración real de share, y
`onInvite` se dispara con el `inviteEmail` actual para que el consumidor decida cómo pedir la
invitación y actualizar la lista `invites`. Mira el [flow Referral](/es/flows/referral/) para ver
este block compuesto dentro de una screen.
