**Status monitoring** es un [block](/es/introduction/blocks/) de aplicaciones: un panel de
salud de servicios con un banner de estado general y una lista de servicios monitoreados, cada
uno con un `Indicator` de estado, un `Badge` y texto opcional de uptime / último incidente.
Compone los primitives `Badge`, `Divider` e `Indicator`, y los tokens `--md-*`.

## Props

| Prop | Tipo | Default | Descripción |
|------|------|---------|-------------|
| `title` | `string` | `'Estado del sistema'` | Título del encabezado del panel |
| `services` | `ServiceItem[]` | 4 servicios de muestra | Servicios monitoreados a renderizar |

### `ServiceItem`

| Campo | Tipo | Descripción |
|-------|------|--------------|
| `id` | `string` | Id único del servicio, usado como clave de la lista |
| `name` | `string` | Nombre del servicio |
| `status` | `'operational' \| 'degraded' \| 'down'` | Estado de salud — define el color del `Indicator`/`Badge` (`operational` → success, `degraded` → warning, `down` → error) y agrega un pulso al indicador cuando es `down` |
| `statusLabel` | `string` | Texto de estado mostrado en el `Badge` |
| `uptime` | `string` | Porcentaje de uptime ya formateado (oculto si se omite) |
| `lastIncident` | `string` | Texto resumen del último incidente (oculto si se omite) |

El banner de estado general del encabezado se calcula automáticamente a partir del peor estado
encontrado en `services` (`down` > `degraded` > `operational`).

## Uso

```tsx
import { StatusMonitoring } from './blocks/StatusMonitoring'

<StatusMonitoring
  title="Estado del sistema"
  services={[
    { id: 'api', name: 'API', status: 'operational', statusLabel: 'Operativo', uptime: '99.98%', lastIncident: 'Sin incidentes en 90 días' },
    { id: 'payments', name: 'Pagos', status: 'degraded', statusLabel: 'Degradado', uptime: '98.40%', lastIncident: 'Incidente hace 2 horas' },
  ]}
/>
```

La API de props es idéntica en los cuatro frameworks.
