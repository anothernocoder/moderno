**Stacked List** es un [block](/es/introduction/blocks/) de aplicación: una lista apilada
verticalmente de registros donde cada fila es su propia tarjeta, con más separación y peso
visual que una fila de lista simple. Cada tarjeta muestra un avatar, texto primario/secundario,
un badge de estado opcional, meta al final y una acción de fila. Compone los primitives `Card`,
`Avatar`, `Badge` y `Button`, y los tokens `--md-*`. Lo copias a tu repo con el CLI y lo editas
libremente.

## Props

| Prop | Tipo | Default | Descripción |
|------|------|---------|-------------|
| `items` | `StackedListItem[]` | 3 tickets de ejemplo | Los registros a renderizar |
| `onItemAction` | `(item: StackedListItem) => void` | — | Se llama al hacer clic en el botón `⋯` de un registro |

Forma de `StackedListItem`:

| Campo | Tipo | Descripción |
|-------|------|--------------|
| `id` | `string` | Id único del registro |
| `title` | `string` | Texto primario |
| `subtitle` | `string` | Texto secundario, bajo el título |
| `avatarInitials` | `string` | Iniciales de respaldo para el `Avatar` |
| `avatarSrc` | `string` | URL de imagen para el `Avatar` |
| `status` | `string` | Etiqueta de estado mostrada en el `Badge` (se omite para ocultar el badge) |
| `statusVariant` | `'neutral' \| 'solid' \| 'success' \| 'warning' \| 'error' \| 'info'` | Variante de color del `Badge` para el estado |
| `meta` | `string` | Texto meta al final, p. ej. una marca de tiempo relativa |

## Uso

```tsx
import { StackedList } from './blocks/StackedList'

<StackedList
  items={[
    { id: 'T-1', title: 'No puede exportar el reporte', subtitle: 'Marina Ortiz · Soporte', avatarInitials: 'MO', status: 'Abierto', statusVariant: 'warning', meta: 'Hace 12 min' },
    { id: 'T-2', title: 'Solicita cambio de plan', subtitle: 'Diego Salazar · Facturación', avatarInitials: 'DS', status: 'Resuelto', statusVariant: 'success', meta: 'Hace 1 h' },
  ]}
  onItemAction={(item) => abrirMenuDeRegistro(item)}
/>
```

La API de props es idéntica en los cuatro frameworks. En Vue, `onItemAction` es el evento
`@item-action`; en los demás es una prop callback.
