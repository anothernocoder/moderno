**Blog Sections** is a marketing [block](/introduction/blocks/): a post listing/preview section
with a centered kicker/title/subtitle header above a responsive grid of post cards, each with an
optional category badge, title, excerpt, and author. It composes the `Card`, `Avatar`, and `Badge`
primitives and the `--md-*` tokens. You copy it into your repo with the CLI and edit it freely.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `kicker` | `string` | `'Blog'` | Small text above the title |
| `title` | `string` | `'Últimas publicaciones'` | Section title |
| `subtitle` | `string` | `'Notas del equipo sobre diseño, ingeniería y producto.'` | Short intro text |
| `posts` | `BlogPostPreview[]` | Three sample posts | The post preview cards |

A `BlogPostPreview` is `{ title: string; excerpt: string; author: string; date: string; avatarSrc?: string; initials?: string; category?: string }`.
Each item renders as a `Card`: an optional `Badge` for `category` at the top, the post title as an
`<h3>`, the excerpt as a paragraph, and a footer with `Avatar` + author name + date. When
`avatarSrc` is absent, `Avatar` falls back to `initials`.

## Usage

```tsx
import { BlogSections } from './blocks/BlogSections'

<BlogSections
  kicker="Blog"
  title="Latest posts"
  subtitle="Notes from the team on design, engineering, and product."
  posts={[
    {
      title: 'How we scaled our design system to four frameworks',
      excerpt: 'Headless components, one API, shared styles.',
      author: 'Jamie Chen',
      date: 'Mar 12, 2025',
      initials: 'JC',
      category: 'Engineering',
    },
  ]}
/>
```

The props API is identical across all four frameworks.

The block renders a centered header (kicker, title, subtitle) above a responsive grid of `Card`s
built from `posts`, one card per post. The grid uses `repeat(auto-fit, minmax(280px, 1fr))`, so
cards reflow from three columns down to one as the viewport narrows.
