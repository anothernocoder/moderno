**Auth** es el primer [flow](/es/introduction/blocks/) de Moderno (ADR-0005): un conjunto
conectado de [screens](/es/introduction/blocks/) que forman un solo recorrido de usuario. Entrega
**cinco screens presentacionales** — sign-in, registro, olvidé mi contraseña, restablecer
contraseña y verificación — más un ensamblaje de ejemplo liviano que las conecta con estado
local: un toggle sign-in↔registro, una rama de "olvidé mi contraseña" que verifica un código
antes de dejar elegir una nueva contraseña, y un correo que se transporta desde
registro/olvidé-mi-contraseña hacia la screen de verificación.

Una **screen** es presentacional: recibe un contrato de wiring tipado (`values`, `onChange`,
`onSubmit`, más callbacks de navegación específicos del flow como `onSignUp`/`onSignIn`/`onBack`)
y no tiene **ni estado interno ni router** — el consumidor decide cómo guardar los valores y
navegar. El **ejemplo** (`Auth.example.{ext}`) es un ensamblaje liviano y ejecutable que arma las
cinco screens usando el estado local propio de cada framework (`useState` / `reactive` / runes /
signals) y una variable `step` local — no es el producto, es un punto de partida copy-paste. El
demo en vivo de abajo ejecuta el ejemplo a través de un único prop `step` (string serializable —
el límite del island de Astro serializa a JSON, así que solo strings simples cruzan esa
frontera); todo lo demás es estado propio del ejemplo.

## Instalación

Traer el flow `auth` copia las cinco screens que compone (`sign-in`, `sign-up`,
`forgot-password`, `reset-password`, `verify`) **más** el ensamblaje de ejemplo. El CLI resuelve
`composes` de forma recursiva y deduplica por nombre — trae cualquier screen individual si es lo
único que quieres.

<Tabs syncKey="framework">
<TabItem label="React">
```sh
npx @moderno/registry add auth --framework react
```
</TabItem>
<TabItem label="Vue">
```sh
npx @moderno/registry add auth --framework vue
```
</TabItem>
<TabItem label="Svelte">
```sh
npx @moderno/registry add auth --framework svelte
```
</TabItem>
<TabItem label="Solid">
```sh
npx @moderno/registry add auth --framework solid
```
</TabItem>
</Tabs>

Esto copia `screens/applications/{sign-in,sign-up,forgot-password,reset-password,verify}/*.{ext}`
y `flows/auth/Auth.example.{ext}` dentro de `--dest` (por defecto `./src/blocks`), preservando esa
estructura relativa para que los imports del ejemplo hacia cada screen sigan funcionando. Pasa
`--no-example` para copiar solo las screens compuestas:

```sh
npx @moderno/registry add auth --framework react --no-example
```

¿Solo quieres una screen, sin el contexto del flow? Agrégala directamente:

```sh
npx @moderno/registry add sign-up --framework react
```

Tras copiarlo, instala el primitive `@moderno/<framework>` e
[importa los estilos](/es/introduction/installation/). A partir de ahí el código — incluyendo la
navegación y el estado — es tuyo.

## Props

### SignIn

| Prop | Tipo | Descripción |
|------|------|-------------|
| `values` | `{ email: string; password: string; remember: boolean }` | Valores controlados de los campos |
| `onChange` | `(values) => void` | Se llama con el siguiente `values` en cada cambio de campo |
| `onSubmit` | `(values) => void` | Se llama con `values` al enviar el formulario |
| `onForgotPassword` | `() => void` | Handler del enlace "¿Olvidaste tu contraseña?" |
| `onSignUp` | `() => void` | Handler del enlace "Regístrate" |
| `error` | `string` | Error del servidor mostrado sobre el formulario |
| `submitting` | `boolean` | Deshabilita el formulario y cambia la etiqueta del submit mientras es `true` |
| `title` | `string` | Texto del encabezado (default `Inicia sesión`) |
| `description` | `string` | Texto del subtítulo |

| `description` | `string` | Texto del subtítulo |

Ninguna screen tiene estado interno ni defaults para `values` — de eso se encarga el ensamblaje
de ejemplo (o tu propio código). Las cinco componen los primitives Input, Button y (sign-in/
sign-up) Checkbox/Divider, y los tokens `--md-*`; sin clases utilitarias legacy.

## Uso

```tsx
import { useState } from 'react'
import { SignIn, type SignInValues } from './screens/applications/sign-in/SignIn'

function App() {
  const [values, setValues] = useState<SignInValues>({ email: '', password: '', remember: false })

  return (
    <SignIn
      values={values}
      onChange={setValues}
      onSubmit={(v) => fetch('/api/sign-in', { method: 'POST', body: JSON.stringify(v) })}
      onForgotPassword={() => router.push('/forgot-password')}
      onSignUp={() => router.push('/sign-up')}
    />
  )
}
```

Conectar tu propio router/estado es justamente el punto: ninguna screen asume que existe uno.
Revisa `Auth.example.tsx` (y sus hermanos `.vue`/`.svelte`/`.solid.tsx`) para ver el recorrido
completo — una unión `step` local que cubre las cinco screens, un correo que se transporta desde
registro/olvidé-mi-contraseña hacia verificación, y un flag `verifyPurpose` que decide si un
código correcto te devuelve a sign-in (cuenta nueva) o te avanza a restablecer contraseña
(recuperación).
