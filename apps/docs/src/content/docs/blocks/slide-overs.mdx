**Slide-overs** is an application [block](/introduction/blocks/): a side drawer for detail/edit
flows. A trigger button opens a [`Sheet`](/components/sheet/) that slides in from the right with
a title, a compact name/email/role form, and a footer with Cancel/Save actions. It composes the
`Sheet`, `Input`, `Button`, and `Divider` primitives plus the `--md-*` tokens. The panel's
open/close state is the `Sheet` primitive's own controlled `open`/`onOpenChange` API — there's no
bespoke overlay, portal, or focus-trap logic in the block itself. You copy it into your repo with
the CLI and edit it freely.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `triggerLabel` | `string` | `'Editar contacto'` | Label of the button that opens the sheet |
| `title` | `string` | `'Editar contacto'` | Sheet title |
| `nameLabel` | `string` | `'Nombre'` | Label of the name field |
| `emailLabel` | `string` | `'Correo'` | Label of the email field |
| `roleLabel` | `string` | `'Rol'` | Label of the role field |
| `cancelLabel` | `string` | `'Cancelar'` | Label of the footer's cancel button |
| `saveLabel` | `string` | `'Guardar cambios'` | Label of the footer's save button |
| `closeLabel` | `string` | `'Cerrar'` | `aria-label` of the sheet's corner close button |
| `defaultValues` | `Partial<{ name, email, role }>` | sample record | Initial form values, merged over the built-in sample record |
| `onSave` | `(values: { name, email, role }) => void` | — | Called with the current form values when "Save" is clicked, before the sheet closes |
| `onCancel` | `() => void` | — | Called when "Cancel" is clicked, before the form resets and the sheet closes |

## Usage

```tsx
import { SlideOvers } from './blocks/SlideOvers'

<SlideOvers
  triggerLabel="Edit member"
  title="Edit member"
  defaultValues={{ name: 'Ada Lovelace', email: 'ada@acme.com', role: 'Engineer' }}
  onSave={(values) => updateMember(values)}
/>
```

The props API is identical across all four frameworks. In Vue, `onSave`/`onCancel` are the
`@save`/`@cancel` events; in the others they're callback props.
