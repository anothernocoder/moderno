**Marketing Stats** is a marketing [block](/introduction/blocks/): a metric/number highlight
section with a centered kicker/title/subtitle header above a row of big numbers and labels. It's
the marketing variant of a stats block — distinct from the applications [`Stats`](/blocks/stats/)
block, which is a `Card`+`Badge` KPI dashboard widget. This one has no cards or badges: it's a
bigger, editorial "brag row" meant for a landing page, composed purely from the `--md-*` tokens.
You copy it into your repo with the CLI and edit it freely.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `kicker` | `string` | `'En números'` | Small text above the title |
| `title` | `string` | `'El impacto de Moderno'` | Section title |
| `subtitle` | `string` | `'Un solo design system, cuatro frameworks, resultados medibles.'` | Short intro text |
| `items` | `StatItem[]` | Four sample stats | The metric items |

A `StatItem` is `{ value: string; label: string }`. Each item renders as a big serif number above
a smaller uppercase label, centered, with no `Card` or `Badge` wrapper.

## Usage

```tsx
import { MarketingStats } from './blocks/MarketingStats'

<MarketingStats
  kicker="By the numbers"
  title="Moderno's impact"
  subtitle="One design system, four frameworks, measurable results."
  items={[
    { value: '4', label: 'Frameworks supported' },
    { value: '30+', label: 'Components' },
    { value: '100%', label: 'Shared tokens' },
    { value: '0', label: 'Extra UI dependencies' },
  ]}
/>
```

The props API is identical across all four frameworks.

The block renders a centered header (kicker, title, subtitle) above a centered, wrapping row of
stat items built from `items`, capped at a max-width of `960px`. Items use `flex-wrap` with a
`48px` gap, so they reflow onto multiple lines as the viewport narrows.
