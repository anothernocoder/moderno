**EmptyStates** is an application [block](/introduction/blocks/): the classic "no data yet"
pattern shown in place of an empty table, list, or section — a bordered icon placeholder, a
heading, a short description, and a primary "create your first X" CTA. There's no illustration
asset — the icon is a bordered box like Features' item icons, consistent with how Moderno's
blocks avoid binary image assets. It composes the `Button` primitive and the `--md-*` tokens.
You copy it into your repo with the CLI and edit it freely.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `icon` | `string` | `'📭'` | Emoji/glyph rendered inside the bordered icon box |
| `title` | `string` | `'Aún no tienes elementos'` | Heading below the icon |
| `description` | `string` | `'Cuando crees tu primer elemento, aparecerá aquí.'` | Supporting text |
| `actionLabel` | `string` | `'Crear el primero'` | Label for the primary `Button` CTA |
| `onAction` | `() => void` | — | Called when the primary CTA is clicked |
| `secondaryLabel` | `string` | `undefined` | Optional label for a secondary text link |
| `secondaryHref` | `string` | `undefined` | Optional `href` for the secondary link |

`secondaryLabel`/`secondaryHref` render as a plain underlined `<a>` next to the primary button —
useful for a "learn more" or "import instead" escape hatch. It only renders when both are provided.

## Usage

```tsx
import { EmptyStates } from './blocks/EmptyStates'

// Default: nothing created yet
<EmptyStates
  title="No projects yet"
  description="Create your first project to get started."
  actionLabel="Create project"
  onAction={() => openCreateProjectModal()}
/>

// Variant: no results from a search/filter
<EmptyStates
  icon="🔍"
  title="No results found"
  description="Try adjusting your search or filters."
  actionLabel="Clear search"
  onAction={() => clearSearch()}
/>
```

The props API is identical across all four frameworks. In Vue and Svelte, `onAction` is a
`@action` event / `onAction` callback prop respectively, mirroring how the other application
blocks (like [Table](/blocks/table/)) expose their callbacks.
