**Panels** is an application [block](/introduction/blocks/): generic panel surfaces for
grouping content within a screen. It pairs a header (title + description) with a responsive
grid of bordered panel surfaces, each with a heading, a body slot, and an optional footer
action. It composes the `Card` and `Button` primitives and the `--md-*` tokens — no new
primitives and no bespoke interaction logic, just a copy-paste layout you can drop content into.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `title` | `string` | `'Ajustes de la cuenta'` | Header heading |
| `description` | `string` | Sample copy | Header subheading (hidden if empty) |
| `panels` | `PanelItem[]` | 4 sample panels | The panel surfaces rendered in the grid |
| `onPanelAction` | `(panel: PanelItem) => void` | — | Called when a panel's footer button is clicked (React/Solid/Svelte); emits `panelAction` in Vue |

`PanelItem` shape:

| Field | Type | Description |
|-------|------|--------------|
| `id` | `string` | Unique panel id / list key |
| `title` | `string` | Panel heading, rendered via `Card`'s `title` slot |
| `body` | `string` | Panel body text |
| `actionLabel` | `string?` | Label for the panel's footer action button (hidden if omitted) |

## Usage

```tsx
import { Panels } from './blocks/Panels'

<Panels
  title="Workspace settings"
  panels={[
    { id: 'profile', title: 'Profile', body: 'Update your name, photo and public bio.', actionLabel: 'Edit profile' },
    { id: 'security', title: 'Security', body: 'Turn on two-factor auth and review active sessions.', actionLabel: 'Review' },
  ]}
  onPanelAction={(panel) => console.log('action', panel.id)}
/>
```

The props API is identical across all four frameworks (Vue exposes the panel action as a
`panelAction` emit instead of an `onPanelAction` prop, matching Vue's event conventions). Each
panel is a `Card` laid out in a responsive `grid-template-columns: repeat(auto-fit, minmax(220px, 1fr))`
grid, so panels reflow automatically as the container width changes. There's no drag-to-reorder,
collapse/expand, or other interaction logic layered on top; wire that up yourself once you've
copied it into your repo.
