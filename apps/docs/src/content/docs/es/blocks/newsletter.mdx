**Newsletter** es un [block](/es/introduction/blocks/) de marketing: una sección de suscripción por
correo con un encabezado centrado (kicker/título/subtítulo) sobre un formulario de correo en línea.
Compone los primitives `Input` y `Button` y los tokens `--md-*`. Lo copias a tu repo con el CLI y lo
editas libremente.

## Props

| Prop | Tipo | Default | Descripción |
|------|------|---------|-------------|
| `kicker` | `string` | `'Newsletter'` | Texto pequeño sobre el título |
| `title` | `string` | `'Recibe las novedades de Moderno'` | Título de la sección |
| `subtitle` | `string` | `'Un correo al mes con nuevos blocks, primitives y notas de diseño.'` | Texto de introducción corto |
| `placeholder` | `string` | `'tu@email.com'` | Placeholder del campo de correo |
| `buttonLabel` | `string` | `'Suscribirme'` | Label del botón de envío |
| `defaultEmail` | `string` | `''` | Valor inicial del campo de correo |
| `onSubmit` | `(email: string) => void` | — | Se llama con el correo al enviarlo (`preventDefault` ya está manejado) |

## Uso

```tsx
import { Newsletter } from './blocks/Newsletter'

<Newsletter
  kicker="Newsletter"
  title="Mantente al día"
  subtitle="Un correo al mes con nuevos blocks, primitives y notas de diseño."
  placeholder="tu@email.com"
  buttonLabel="Suscribirme"
  onSubmit={(email) => console.log(email)}
/>
```

La API de props es idéntica en los cuatro frameworks (en Vue, `onSubmit` es el evento `@submit` en
lugar de un prop).

El block renderiza un encabezado centrado (kicker, título, subtítulo) sobre un formulario en una
sola fila con un `Input` de correo y un `Button` de envío. Maneja internamente el estado de su
campo y llama a `onSubmit` con el correo actual; conecta ese callback a tu propia lógica de
request.
