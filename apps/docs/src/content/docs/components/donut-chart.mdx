**Donut chart** plots `{label, value}` data as one SVG arc per datum, sized
proportionally to its share of the total ŌĆö in input order, not re-sorted by size. It
builds on the same **chart-core** geometry model as [Line chart](/components/line-chart/)
(ADR-0004) ŌĆö `@moderno/chart-core`, built on `d3-scale` + `d3-shape`'s `pie`/`arc`
generators ŌĆö computed once and rendered as SVG by each framework package. Slices cycle
through five `--md-*` semantic color tokens (`primary`, `info`, `success`, `warning`,
`error`) since the token set has no dedicated categorical chart palette. A
**closed-prop** primitive (ADR-0003) ŌĆö pass `data` and size props, no compositional
anatomy.

## Usage

<DonutChart
  data={[

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `data` | `{ label: string; value: number }[]` | ŌĆö | The slices to plot, in display order (not re-sorted). |
| `width` | `number` | `480` | Intrinsic SVG width in px. The chart scales to 100% of its container via CSS; `width`/`height` set the aspect ratio (`viewBox`). |
| `height` | `number` | `240` | Intrinsic SVG height in px. |
| `innerRadiusRatio` | `number` | `0.6` | Donut hole radius as a ratio of the outer radius; `0` renders a full pie. |
| `label` | `string` | ŌĆö | Accessible label read by screen readers in place of the visual chart. |
