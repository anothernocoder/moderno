**Contenedores de lista** es un [block](/es/introduction/blocks/) de aplicaciones: un
contenedor especializado para listas que combina un encabezado fijo con un cuerpo con scroll
— un envoltorio de altura acotada con `overflow-y: auto` — y un footer con una línea de resumen
y un botón de acción. Compone los primitives `Avatar`, `Badge`, `Button` y `Divider` y los
tokens `--md-*` — sin primitives nuevos ni lógica de interacción a medida, solo un layout
copy-paste en el que puedes insertar una lista.

## Props

| Prop | Tipo | Default | Descripción |
|------|------|---------|-------------|
| `title` | `string` | `'Tareas del sprint'` | Encabezado del panel |
| `description` | `string` | Texto de ejemplo | Subtítulo del encabezado (se oculta si está vacío) |
| `items` | `ListContainerItem[]` | 6 filas de ejemplo | Las filas renderizadas en el cuerpo con scroll |
| `maxBodyHeight` | `string` | `'280px'` | `max-height` CSS aplicado al cuerpo con scroll |
| `footerLabel` | `string` | `'Ver todas las tareas'` | Etiqueta del botón de acción del footer |
| `onFooterAction` | `() => void` | — | Se invoca al hacer click en el botón del footer (React/Solid/Svelte); en Vue emite `footerAction` |

Forma de `ListContainerItem`:

| Campo | Tipo | Descripción |
|-------|------|--------------|
| `id` | `string` | Id único de la fila / key de lista |
| `title` | `string` | Texto principal de la fila |
| `subtitle` | `string?` | Texto secundario de la fila |
| `meta` | `string?` | Texto meta final (p. ej. una fecha relativa) |
| `avatarSrc` | `string?` | URL de la imagen del avatar |
| `avatarInitials` | `string?` | Iniciales de respaldo del avatar |
| `status` | `string?` | Etiqueta del badge de estado |
| `statusVariant` | `'neutral' \| 'solid' \| 'success' \| 'warning' \| 'error' \| 'info'` | Variante del badge de estado |

## Uso

```tsx
import { ListContainers } from './blocks/ListContainers'

<ListContainers
  title="Tareas abiertas"
  maxBodyHeight="320px"
  items={[
    { id: '1', title: 'Publicar el changelog', subtitle: 'Docs', status: 'En curso', statusVariant: 'info' },
    { id: '2', title: 'Arreglar bug de scroll en Safari', subtitle: 'Ingeniería', status: 'Bloqueada', statusVariant: 'error' },
  ]}
  footerLabel="Ver todas las tareas"
  onFooterAction={() => console.log('ver todas')}
/>
```

La API de props es idéntica en los cuatro frameworks (Vue expone la acción del footer como un
emit `footerAction` en lugar de una prop `onFooterAction`, siguiendo las convenciones de eventos
de Vue). El cuerpo es un bloque simple de filas separadas por `Divider` dentro de un envoltorio
cuyo `max-height` se toma de `maxBodyHeight` con `overflow-y: auto` — una vez que las filas
superan esa altura el cuerpo hace scroll mientras el encabezado y el footer permanecen fijos. No
hay virtualización, scroll infinito ni otra lógica de interacción por encima; agrégala tú mismo
una vez que lo hayas copiado a tu repo.
