**Order Summaries** es un [block](/es/introduction/blocks/) de ecommerce: un panel de
resumen de pedido de solo lectura que lista líneas de productos — cada una con título,
cantidad y total de línea — seguido de filas de subtotal, envío e impuestos, y un total
final. Se muestra típicamente junto a un formulario de checkout o una confirmación de
pedido. Compone el primitive `Divider` junto con los tokens `--md-*`. Lo copias a tu repo
con el CLI y lo editas libremente.

Este block es puramente presentacional: a diferencia del block
[Shopping Cart](/es/blocks/shopping-cart/), no hay steppers de cantidad ni acciones de
quitar — las cantidades y totales se calculan una vez a partir del prop `items` y se
renderizan como texto estático.

## Props

| Prop | Tipo | Default | Descripción |
|------|------|---------|-------------|
| `heading` | `string` | `'Resumen del pedido'` | Encabezado sobre el panel (omítelo para ocultarlo) |
| `items` | `OrderSummariesItem[]` | 3 líneas de ejemplo | Líneas del pedido — cada una `{ id, title, price, quantity? }` |
| `emptyMessage` | `string` | `'No hay productos en el pedido.'` | Mensaje mostrado cuando `items` está vacío |
| `subtotalLabel` | `string` | `'Subtotal'` | Etiqueta de la fila de subtotal |
| `shippingLabel` | `string` | `'Envío'` | Etiqueta de la fila de envío |
| `shipping` | `number` | `15000` | Costo de envío sumado al total; `0` muestra `freeShippingLabel` en su lugar |
| `freeShippingLabel` | `string` | `'Gratis'` | Texto mostrado en la fila de envío cuando `shipping` es `0` |
| `taxLabel` | `string` | `'Impuestos'` | Etiqueta de la fila de impuestos |
| `taxRate` | `number` | `0.19` | Tasa de impuesto aplicada al subtotal para calcular la fila de impuestos (p. ej. `0.19` = 19%) |
| `totalLabel` | `string` | `'Total'` | Etiqueta de la fila de total final |

## Uso

```tsx
import { OrderSummaries } from './blocks/OrderSummaries'

<OrderSummaries
  items={[
    { id: 'silla', title: 'Silla Roble Nórdica', price: 389000, quantity: 1 },
    { id: 'lampara', title: 'Lámpara de Arco', price: 128000, quantity: 2 },
  ]}
  shipping={0}
/>
```

La API de props es idéntica en los cuatro frameworks. El `price` de cada línea es un
`number` simple (formateado internamente con `toLocaleString`); el subtotal es la suma de
`price * quantity` en todas las líneas, calculado una vez a partir del prop `items` — no
hay estado editable local, así que pasar nuevos `items` vuelve a renderizar los totales
correspondientes. El impuesto se deriva del subtotal mediante `taxRate`, así que puedes
pasar un `taxRate` fijo de `0` y aportar tu propio impuesto ya calculado a través de una
`taxLabel`/fila personalizada si tu backend ya devuelve un monto de impuesto en lugar de
una tasa. Sin primitives nuevos, sin controles interactivos — solo el primitive `Divider`
existente compuesto con layout, a diferencia de la edición de cantidad del block
[Shopping Cart](/es/blocks/shopping-cart/).
