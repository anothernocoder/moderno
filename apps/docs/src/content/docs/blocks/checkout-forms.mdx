**Checkout Forms** is an ecommerce [block](/introduction/blocks/): a checkout page with a
shipping-address section and a payment section grouped into a multi-section form,
separated by a divider, with a footer submit action. It composes the `Input`, `Select`,
`Divider`, and `Button` primitives plus the `--md-*` tokens. You copy it into your repo
with the CLI and edit it freely.

This is a static UI-only block: the card number, expiry, and CVC fields are plain text
inputs styled like the rest of the form вЂ” there is no masking, no validation, no
luhn-check, and no real payment processing. Submitted values only ever live in local
component state until you wire up `onCheckout` yourself.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `heading` | `string` | `'Finalizar compra'` | Heading above the form (omit to hide it) |
| `shippingTitle` | `string` | `'DirecciГіn de envГ­o'` | Title for the shipping-address section |
| `shippingDescription` | `string` | `'A dГіnde enviamos tu pedido.'` | Description under the shipping-address title |
| `paymentTitle` | `string` | `'MГ©todo de pago'` | Title for the payment section |
| `paymentDescription` | `string` | `'Datos de la tarjeta para procesar el cobro.'` | Description under the payment title |
| `defaultValues` | `Partial<CheckoutFormsValues>` | sample shipping data, empty card fields | Initial field values, merged over the defaults |
| `submitLabel` | `string` | `'Confirmar pedido'` | Submit button text |
| `onCheckout` | `(values: CheckoutFormsValues) => void` | вЂ” | Called on form submit with the current field values |

`CheckoutFormsValues` is `{ fullName, email, address, city, postalCode, country, cardName, cardNumber, cardExpiry, cardCvc }` вЂ” all `string`.

## Usage

```tsx
import { CheckoutForms } from './blocks/CheckoutForms'

<CheckoutForms
  onCheckout={(values) => checkout.submit(values)}
/>
```

The props API is identical across all four frameworks. In Vue, `onCheckout` is the
`@checkout` event; in the others it's a callback prop. Every field is a controlled input
backed by local component state, seeded from `defaultValues` on mount вЂ” the same pattern
used by the [Form Layouts](/blocks/form-layouts/) application block. The country field
uses the `Select` primitive; every other field, including the card number, expiry, and
CVC, is a plain `Input`. No new primitives, no bespoke interaction logic, no card
masking or validation вЂ” that's left for you to add if you need it, and no data ever
leaves local state unless you wire up `onCheckout` to your own backend.
