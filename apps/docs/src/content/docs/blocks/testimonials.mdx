**Testimonials** is a marketing [block](/introduction/blocks/): a social-proof section with a
centered kicker/title/subtitle header above a grid of customer quotes, each with an author name,
role, and avatar. It composes the `Card` and `Avatar` primitives and the `--md-*` tokens. You copy
it into your repo with the CLI and edit it freely.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `kicker` | `string` | `'Testimonios'` | Small text above the title |
| `title` | `string` | `'Lo que dicen nuestros clientes'` | Section title |
| `subtitle` | `string` | `'Equipos de todos los tamaños construyen con Moderno.'` | Short intro text |
| `items` | `TestimonialItem[]` | Three sample testimonials | The quote cards |

A `TestimonialItem` is `{ quote: string; name: string; role?: string; avatarSrc?: string; initials?: string }`.
Each item renders as a `Card` with the quote as its body and the author (`Avatar` + name + role) as
its footer. When `avatarSrc` is absent, `Avatar` falls back to `initials`.

## Usage

```tsx
import { Testimonials } from './blocks/Testimonials'

<Testimonials
  kicker="Testimonials"
  title="Loved by teams everywhere"
  subtitle="Here's what our customers have to say."
  items={[
    {
      quote: 'Moderno let us ship our redesign in two weeks flat.',
      name: 'Jamie Chen',
      role: 'Head of Design, Nimbus',
      initials: 'JC',
    },
    {
      quote: 'One API across React, Vue, Svelte, and Solid — huge win for our team.',
      name: 'Alex Rivera',
      role: 'Frontend Lead, Kairos',
      initials: 'AR',
    },
  ]}
/>
```

The props API is identical across all four frameworks.

The block renders a centered header (kicker, title, subtitle) above a responsive grid of `Card`s
built from `items`, one card per testimonial. The grid uses `repeat(auto-fit, minmax(280px, 1fr))`,
so cards reflow from three columns down to one as the viewport narrows.
