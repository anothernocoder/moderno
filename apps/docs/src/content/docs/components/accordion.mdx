**Accordion** groups collapsible sections that reveal their content when activated. A
**compositional** primitive (ADR-0003): it exposes parts that share a single Zag machine —
`Accordion.Root`, `Accordion.Item`, `Accordion.ItemTrigger`, `Accordion.ItemContent`. The
behavior (arrow-key navigation, `Home`/`End`, ARIA `region`, expand/collapse) comes from
the `accordion` machine.

## Usage

## Props (`Accordion.Root`)

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `defaultValue` | `string[]` | `[]` | Initially expanded items (uncontrolled). |
| `value` / `v-model` / `bind:value` | `string[]` | — | Expanded items (controlled). |
| `multiple` | `boolean` | `false` | Allows multiple items open at once. |
| `collapsible` | `boolean` | `false` | Allows closing an already-open item. |
| `onValueChange` | `(value: string[]) => void` | — | Callback on expand/collapse. |

`Accordion.Item` takes `value: string` and `disabled?: boolean`. `Accordion.ItemTrigger` and
`Accordion.ItemContent` need no props: they inherit the item via context.
