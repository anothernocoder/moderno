**Reviews** is an ecommerce [block](/introduction/blocks/): a customer reviews section
pairing an aggregate rating summary — average rating, review count, and a breakdown bar
per star level — with a list of individual review entries (reviewer avatar/name, star
rating, date, and text). It composes the `Avatar`, `Badge`, `Card`, and `Progress`
primitives plus the `--md-*` tokens. You copy it into your repo with the CLI and edit it
freely.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `heading` | `string` | `'Reseñas de clientes'` | Heading above the section (omit to hide it) |
| `averageRating` | `number` | `4.6` | Average rating out of 5, shown as a large number and star row |
| `reviewCount` | `number` | `132` | Total review count — also used to compute each breakdown bar's percentage |
| `breakdown` | `ReviewsBreakdownItem[]` | 5 sample rows | Per-star counts — each `{ stars: number, count: number }`, rendered 5→1 |
| `reviews` | `ReviewEntry[]` | 3 sample reviews | Individual reviews — each with `name`, `rating`, `date`, `text`, and optional `avatarSrc`, `initials`, `verified` |

## Usage

```tsx
import { Reviews } from './blocks/Reviews'

<Reviews
  heading="What customers say"
  averageRating={4.8}
  reviewCount={214}
  breakdown={[
    { stars: 5, count: 160 },
    { stars: 4, count: 40 },
    { stars: 3, count: 10 },
    { stars: 2, count: 3 },
    { stars: 1, count: 1 },
  ]}
  reviews={[
    {
      name: 'Ana López',
      initials: 'AL',
      rating: 5,
      date: 'June 3, 2026',
      text: 'Exceeded my expectations. Sturdy build and a beautiful finish.',
      verified: true,
    },
  ]}
/>
```

The props API is identical across all four frameworks. The summary column renders the
average rating, a star row, the review count, and one `Progress` bar per star level
(5→1) showing what share of reviews landed at that level. The list column renders one
`Card` per review, pairing an `Avatar` (image or initials fallback) and name with an
optional `Badge` for verified purchases, a star row, the date, and the review text. Star
ratings are drawn inline as SVG `path` shapes — the same inline-icon convention used by
the [Product Features](/blocks/product-features/) block — so no new "star rating"
primitive was introduced. No new primitives, no bespoke interaction logic — this block
is purely presentational.
