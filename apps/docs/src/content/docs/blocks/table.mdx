**Table** is an application [block](/introduction/blocks/): a data table with a sortable
order column, per-row selection (indeterminate "select all"), status badges, and a footer
that surfaces bulk actions once rows are selected. It composes the `Checkbox`, `Badge`, and
`Button` primitives and the `--md-*` tokens. You copy it into your repo with the CLI and edit
it freely.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `rows` | `TableRow[]` | 3 sample orders | The rows to render |
| `onExport` | `(selected: TableRow[]) => void` | — | Called when "Export" is clicked, with the selected rows |
| `onCancelSelected` | `(selected: TableRow[]) => void` | — | Called when "Cancel orders" is clicked, with the selected rows |
| `onRowAction` | `(row: TableRow) => void` | — | Called when a row's `⋯` action button is clicked |

`TableRow` shape:

| Field | Type | Description |
|-------|------|--------------|
| `id` | `string` | Unique row id, also used as the selection key |
| `order` | `string` | Order reference, the sortable column |
| `customer` | `string` | Customer name |
| `status` | `string` | Status label shown in the `Badge` |
| `statusVariant` | `'neutral' \| 'solid' \| 'success' \| 'warning' \| 'error' \| 'info'` | `Badge` color variant for the status |
| `total` | `string` | Formatted order total, right-aligned |

## Usage

```tsx
import { Table } from './blocks/Table'

<Table
  rows={[
    { id: 'A-1', order: 'A-1', customer: 'Acme Co', status: 'Shipped', statusVariant: 'success', total: '$120' },
    { id: 'A-2', order: 'A-2', customer: 'Globex', status: 'Pending', statusVariant: 'warning', total: '$40' },
  ]}
  onExport={(rows) => exportOrders(rows)}
  onCancelSelected={(rows) => cancelOrders(rows)}
  onRowAction={(row) => openOrderMenu(row)}
/>
```

The props API is identical across all four frameworks. In Vue, `onExport`/`onCancelSelected`/
`onRowAction` are the `@export`/`@cancel-selected`/`@row-action` events; in the others they're
callback props.
