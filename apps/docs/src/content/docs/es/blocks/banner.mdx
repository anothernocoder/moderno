**Banner** es un [block](/es/introduction/blocks/) de marketing: una franja de anuncio en la
parte superior de la página, con copy y un CTA opcional (como botón o como enlace). Compone
el primitive `Button` y los tokens `--md-*`. Lo copias a tu repo con el CLI y lo editas
libremente.

## Props

| Prop | Tipo | Default | Descripción |
|------|------|---------|-------------|
| `message` | `string` | Copy de ejemplo | El texto del banner |
| `ctaLabel` | `string` | `'Ver blocks'` | Texto del CTA opcional. Pásalo como string vacío para ocultar el CTA |
| `ctaHref` | `string?` | — | Renderiza el CTA como enlace en vez de botón |
| `onCtaClick` | `() => void` | — | Se llama al hacer clic en el botón del CTA (se ignora si `ctaHref` está definido) |
| `dismissible` | `boolean` | `true` | Si se muestra el botón de cierre (×) |
| `onDismiss` | `() => void` | — | Se llama al cerrar el banner |
| `closeLabel` | `string` | `'Cerrar'` | Etiqueta accesible del botón de cierre |

## Uso

```tsx
import { Banner } from './blocks/Banner'

<Banner
  message="Nuevo: los blocks ya están disponibles para React, Vue, Svelte y Solid."
  ctaLabel="Leer la documentación"
  ctaHref="/es/introduction/blocks/"
  onDismiss={() => track('banner_dismissed')}
/>
```

La API de props es idéntica en los cuatro frameworks. En Vue, `onCtaClick` y `onDismiss` son
los eventos `@cta` y `@dismiss`; en los demás son props callback.

El cierre es estado local del componente — el banner se desmonta a sí mismo al cerrarse. Si
necesitas que permanezca cerrado entre cargas de página, persiste eso en `onDismiss` (por
ejemplo en `localStorage`) y evita renderizar el block cuando tu propio estado indique que ya
fue cerrado.
