**Media objects** is an application [block](/introduction/blocks/): a list of rows, each
pairing an [`Avatar`](/components/avatar/) — image or initials fallback — with a text block
(heading, body copy, and an optional meta line). This is the classic "media object" layout
pattern, well suited to comment threads, activity digests, or contact lists. It composes the
`Avatar` primitive and the `--md-*` tokens — no bespoke interaction logic, just a copy-paste
layout.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `title` | `string` | `'Comentarios'` | Panel heading |
| `description` | `string` | Sample copy | Panel subheading (hidden if empty) |
| `items` | `MediaObjectItem[]` | 3 sample comments | The rows to render, each pairing an `Avatar` with a text block |

`MediaObjectItem` shape:

| Field | Type | Description |
|-------|------|--------------|
| `id` | `string` | Unique row id / React key |
| `heading` | `string` | Row heading (e.g. author name) |
| `body` | `string` | Row body copy |
| `meta` | `string?` | Optional trailing meta line (e.g. timestamp) |
| `avatarSrc` | `string?` | `Avatar`'s image `src` |
| `avatarAlt` | `string?` | `Avatar`'s image `alt` |
| `initials` | `string` | `Avatar`'s initials fallback, shown when `avatarSrc` is absent |

## Usage

```tsx
import { MediaObjects } from './blocks/MediaObjects'

<MediaObjects
  title="Comments"
  items={[
    {
      id: 'valentina-rios',
      heading: 'Valentina Ríos',
      body: 'The new checkout flow looks much clearer. Can we apply the same pattern to the orders panel?',
      meta: '10 min ago',
      initials: 'VR',
    },
    {
      id: 'mateo-fernandez',
      heading: 'Mateo Fernández',
      body: 'Nice work on the empty states. I suggest adding an illustrative icon to reinforce the message.',
      meta: '1 h ago',
      initials: 'MF',
    },
  ]}
/>
```

The props API is identical across all four frameworks. `Avatar` is a pure-CSS, closed-prop
primitive (ADR-0003): pass `avatarSrc`/`avatarAlt` for an image, or omit them and it falls back
to `initials`. This block only arranges `Avatar` instances next to a text block — there's no
selection state, expandable replies, or other interaction logic layered on top; add that once
you've copied it into your repo.
