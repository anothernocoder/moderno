**Features** is a marketing [block](/introduction/blocks/): a feature highlight grid with an
icon, title, and description per item. It composes the `Card` primitive and the `--md-*`
tokens. You copy it into your repo with the CLI and edit it freely.

[import the styles](/introduction/installation/). From there, the code is yours.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `kicker` | `string` | `'Características'` | Small text above the title |
| `title` | `string` | `'Todo lo que necesitas para construir rápido'` | Main headline |
| `subtitle` | `string` | `'Primitives, blocks y tokens que funcionan igual en los cuatro frameworks.'` | Supporting text |
| `features` | `{ icon?: string; title: string; description: string }[]` | Six sample features | The feature grid items. `icon` is rendered as-is (an emoji or short glyph) |

## Usage

```tsx
import { Features } from './blocks/Features'

<Features
  kicker="Features"
  title="Everything you need to ship fast"
  subtitle="Primitives, blocks, and tokens that work the same across all four frameworks."
  features={[
    { icon: '⚡', title: 'Fast by default', description: 'No runtime CSS-in-JS, no client-side theming.' },
    { icon: '🔒', title: 'Type-safe', description: 'Every prop and slot is typed end to end.' },
  ]}
/>
```

The props API is identical across all four frameworks.

Each item renders in a `Card`, laid out in a responsive grid
(`repeat(auto-fit, minmax(240px, 1fr))`) that reflows from a multi-column grid down to a
single column on narrow viewports. `icon` is optional — omit it per item for a plain
title/description card.
