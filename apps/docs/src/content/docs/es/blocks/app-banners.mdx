**App Banners** es un [block](/es/introduction/blocks/) de aplicaciones: una pila de franjas de
notificación de sistema a todo el ancho, típicamente en la parte superior del shell de una app
autenticada — una advertencia de impersonación, una cuenta regresiva de fin de prueba, un aviso
de servicio degradado. Cada banner tiene un fondo de color semántico, un icono al inicio, un
mensaje, una acción opcional y un control de cierre opcional. Compone el primitive `Button` y
los tokens `--md-*`.

Este es un block distinto de [Banner](/es/blocks/banner/), que es la franja de anuncio/promo de
marketing en la parte superior de la página.

## Props

| Prop | Tipo | Default | Descripción |
|------|------|---------|-------------|
| `items` | `AppBannerItem[]` | 3 avisos de ejemplo | Los banners a renderizar, apilados de arriba a abajo |
| `onDismiss` | `(id: string) => void` | — | Se llama con el `id` del ítem al hacer clic en su botón de cierre |

Forma de `AppBannerItem`:

| Campo | Tipo | Default | Descripción |
|-------|------|---------|-------------|
| `id` | `string` | — | Id único; también se pasa a `onDismiss` |
| `variant` | `'info' \| 'success' \| 'warning' \| 'error'` | `'info'` | Color semántico de la franja y el icono |
| `icon` | `string` | — | Icono al inicio (emoji o glifo corto), oculto si se omite |
| `message` | `string` | — | Texto del banner |
| `actionLabel` | `string` | — | Etiqueta de la acción final opcional, oculta si se omite |
| `actionHref` | `string` | — | Si se define, la acción se renderiza como enlace en vez de `Button` |
| `onAction` | `() => void` | — | Se llama al hacer clic en el `Button` de acción (se ignora si hay `actionHref`) |
| `dismissible` | `boolean` | `true` | Poner en `false` para avisos persistentes (ej. impersonación) que no se pueden cerrar |
| `closeLabel` | `string` | `'Cerrar'` | Etiqueta accesible del botón de cierre |

## Uso

```tsx
import { AppBanners } from './blocks/AppBanners'

<AppBanners
  items={[
    {
      id: 'impersonation',
      variant: 'warning',
      icon: '🕵️',
      message: 'Estás impersonando a Ana Pérez.',
      actionLabel: 'Salir',
      dismissible: false,
    },
    {
      id: 'trial',
      variant: 'info',
      icon: '⏳',
      message: 'Tu prueba termina en 3 días.',
      actionLabel: 'Actualizar plan',
    },
    {
      id: 'incident',
      variant: 'error',
      icon: '⚠️',
      message: 'El servicio está actualmente degradado.',
    },
  ]}
  onDismiss={(id) => cerrarBanner(id)}
/>
```

La API de props es idéntica en los cuatro frameworks. En Vue, `onDismiss` es el evento
`@dismiss`; en React, Svelte y Solid es la prop callback `onDismiss`.
