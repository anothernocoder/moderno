**Category Preview** es un [block](/es/introduction/blocks/) de ecommerce: un
grid de tiles de categoría — una imagen y una etiqueta cada uno — que enlazan
a las páginas de categoría de una tienda (por ejemplo "Sillas", "Mesas",
"Iluminación"). Layout plano compuesto con los tokens `--md-*` — sin
primitives, sin lógica de interacción a medida. Lo copias a tu repo con el CLI
y lo editas libremente.

## Props

| Prop | Tipo | Default | Descripción |
|------|------|---------|-------------|
| `heading` | `string` | `'Comprar por categoría'` | Encabezado sobre el grid (omítelo para ocultarlo) |
| `categories` | `CategoryPreviewItem[]` | 4 categorías de ejemplo | Tiles a renderizar — cada uno con `id`, `label`, `image` y `href` |
| `onCategoryClick` | `(category: CategoryPreviewItem) => void` | — | Se llama al hacer clic en un tile |

## Uso

```tsx
import { CategoryPreview } from './blocks/CategoryPreview'

<CategoryPreview
  heading="Comprar por categoría"
  categories={[
    { id: 'sillas', label: 'Sillas', image: '/img/sillas.jpg', href: '/categorias/sillas' },
    { id: 'mesas', label: 'Mesas', image: '/img/mesas.jpg', href: '/categorias/mesas' },
  ]}
  onCategoryClick={(category) => trackCategoryClick(category.id)}
/>
```

La API de props es idéntica en los cuatro frameworks. En Vue, `onCategoryClick`
es el evento `@category-click`; en React, Svelte y Solid es una prop callback.
Cada tile es solo un enlace que envuelve una imagen y una etiqueta — no
introduce ningún primitive nuevo.
