**Logos** is a marketing [block](/introduction/blocks/): a "trusted by" logo strip with a small
centered kicker line above a wrapped, centered row of partner or customer logos. It's plain
composition over the `--md-*` tokens — no primitives involved. You copy it into your repo with the
CLI and edit it freely.

[import the styles](/introduction/installation/). From there, the code is yours.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `kicker` | `string` | `'Con la confianza de equipos de todos los tamaños'` | Small text above the logo row |
| `items` | `LogoItem[]` | Six sample companies | The logo slots |

A `LogoItem` is `{ name: string; src?: string; href?: string }`. When `item.href` is set, the slot
is wrapped in an `<a>`. When `item.src` is set, an `<img>` is rendered (muted with `grayscale(1)`
and reduced opacity); otherwise the slot falls back to `item.name` rendered as a muted, bold,
letter-spaced uppercase wordmark placeholder.

## Usage

```tsx
import { Logos } from './blocks/Logos'

<Logos
  kicker="Trusted by teams of every size"
  items={[
    { name: 'Nimbus' },
    { name: 'Kairos' },
    { name: 'Vela', href: 'https://example.com' },
    { name: 'Orbit', src: '/logos/orbit.svg' },
  ]}
/>
```

The props API is identical across all four frameworks.

The block renders a centered kicker paragraph above a `flex-wrap` row of logo slots, capped at a
max-width of 960px and centered via margin auto. The row stays centered and wraps onto additional
lines as the viewport narrows, so it works with any number of logos.
