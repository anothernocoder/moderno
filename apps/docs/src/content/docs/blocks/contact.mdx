**Contact** is a marketing [block](/introduction/blocks/): a contact section with intro copy and
a contact info list on the left, and a name/email/subject/message form on the right. It composes
the `Input`, `Textarea`, and `Button` primitives and the `--md-*` tokens. You copy it into your
repo with the CLI and edit it freely.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `eyebrow` | `string` | `'Contacto'` | Small label above the title |
| `title` | `string` | `'┬┐Hablamos?'` | Section title |
| `description` | `string` | `'Escr├Łbenos y te respondemos en menos de 24 horas.'` | Short intro text |
| `contactInfo` | `ContactInfoItem[]` | Email/phone/office samples | Contact details list |
| `nameLabel` | `string` | `'Nombre'` | Name field label |
| `emailLabel` | `string` | `'Correo'` | Email field label |
| `subjectLabel` | `string` | `'Asunto'` | Subject field label |
| `messageLabel` | `string` | `'Mensaje'` | Message field label |
| `submitLabel` | `string` | `'Enviar mensaje'` | Submit button label |
| `defaultValues` | `Partial<ContactFormValues>` | `{}` | Initial values for the form fields |
| `onSubmit` | `(values: ContactFormValues) => void` | ŌĆö | Called with the form values on submit (`preventDefault` is already handled) |

A `ContactInfoItem` is `{ label: string; value: string; href?: string }`. `ContactFormValues` is
`{ name: string; email: string; subject: string; message: string }`.

## Usage

```tsx
import { Contact } from './blocks/Contact'

<Contact
  eyebrow="Contact"
  title="Let's talk"
  description="Send us a message and we'll get back to you within a day."
  contactInfo={[
    { label: 'Email', value: 'hello@acme.com', href: 'mailto:hello@acme.com' },
    { label: 'Phone', value: '+1 555 010 1234', href: 'tel:+15550101234' },
  ]}
  nameLabel="Name"
  emailLabel="Email"
  subjectLabel="Subject"
  messageLabel="Message"
  submitLabel="Send message"
  onSubmit={(values) => console.log(values)}
/>
```

The props API is identical across all four frameworks (in Vue, `onSubmit` is the `@submit` event
instead of a prop).

The block renders an intro column (eyebrow, title, description, contact info) next to a form with
name and email side by side, a subject field, a message textarea, and a submit button. It manages
its own field state internally and calls `onSubmit` with the current values; wire that callback up
to your own request logic.
