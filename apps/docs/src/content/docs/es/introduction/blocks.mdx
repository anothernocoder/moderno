Moderno distribuye en **dos canales** (ADR-0001):

- **Primitives** → paquetes npm (`@moderno/react`, etc.). Versionados; los arreglos se
  propagan al centro. Son los [componentes](/es/components/button/).
- **Blocks** → **registry copy-paste**. Composiciones de layout (Hero, Pricing, app shell)
  que copias a tu repo y editas libremente. No se actualizan solos — ese es el trade por la
  libertad de layout.

## El CLI

```sh
# Ver los blocks disponibles
npx @moderno/registry list

# Copiar un block a tu proyecto, para tu framework
npx @moderno/registry add hero --framework react --dest ./src/blocks
```

El comando copia el archivo del block (para el framework elegido) a `--dest`
(por defecto `./src/blocks`). A partir de ahí el código es **tuyo**: edítalo a gusto.

Tras copiarlo, asegúrate de tener instalado el paquete primitive correspondiente
(`@moderno/<framework>`) y de [importar los estilos](/es/introduction/installation/).

## Bloques disponibles

Cada block tiene su propia página con demo en vivo (los cuatro frameworks), comando de
instalación y referencia de props en la sección **Blocks** del menú:

- [Hero](/es/blocks/hero/) — marketing: hero above-the-fold.
- [Banner](/es/blocks/banner/) — marketing: franja de anuncio en la parte superior, cerrable.
- [Pricing](/es/blocks/pricing/) — marketing: sección de comparación de planes.
- [Header](/es/blocks/header/) — marketing: encabezado de sitio / navegación con logo, enlaces y CTA(s).
- [Features](/es/blocks/features/) — marketing: grid de características con icono, título y descripción.
- [Content](/es/blocks/content/) — marketing: sección de contenido/prosa general con encabezados y texto.
- [Careers](/es/blocks/careers/) — marketing: sección de carreras/vacantes con título de rol, ubicación y CTA de aplicar.
- [Footer](/es/blocks/footer/) — marketing: pie de sitio con columnas de navegación, enlaces legales y enlaces sociales.
- [Contact](/es/blocks/contact/) — marketing: sección de contacto con datos de contacto y un formulario de nombre/correo/asunto/mensaje.
- [FAQ](/es/blocks/faq/) — marketing: sección de preguntas frecuentes con un encabezado centrado sobre una lista de preguntas y respuestas.
- [Testimonials](/es/blocks/testimonials/) — marketing: sección de prueba social con tarjetas de cita, autor y avatar.
- [Logos](/es/blocks/logos/) — marketing: franja de logos de confianza con kicker opcional.
- [Team](/es/blocks/team/) — marketing: sección de equipo con tarjetas de avatar, nombre, rol y bio.
- [CTA](/es/blocks/cta/) — marketing: sección de titular + botón de llamado a la acción, más simple que Hero.
- [Blog header](/es/blocks/blog-header/) — marketing: kicker/título/subtítulo centrados sobre una fila de chips de categoría para el índice de un blog.
- [Blog sections](/es/blocks/blog-sections/) — marketing: encabezado centrado sobre una grilla responsiva de tarjetas de preview de posts (título, extracto, autor, categoría).
- [Blog post header](/es/blocks/blog-post-header/) — marketing: encabezado de artículo individual con badge de categoría, título grande, subtítulo y fila de meta de autor/fecha/lectura.
- [Newsletter](/es/blocks/newsletter/) — marketing: sección de suscripción por correo con un encabezado centrado sobre un formulario en línea.
- [Marketing Stats](/es/blocks/marketing-stats/) — marketing: fila de métricas/números destacados para landing pages (números grandes en serif, sin cards).
- [Contact Modal](/es/blocks/contact-modal/) — marketing: captura de contacto/leads con formulario compacto y un trigger de contacto directo basado en Dialog.
- [404](/es/blocks/not-found/) — marketing: sección de error 404 con un numeral serif grande, mensaje y enlaces CTA de vuelta al sitio.
- [Stats](/es/blocks/stats/) — aplicación: fila de KPIs.
- [Table](/es/blocks/table/) — aplicación: tabla de datos con selección y acciones en lote.
- [Form Layouts](/es/blocks/form-layouts/) — aplicación: layout de página de formulario de configuración/edición de perfil con secciones agrupadas, divisores y un grid responsivo de campos.
- [Product Card](/es/blocks/product-card/) — ecommerce: tarjeta de producto.
- [Product Details](/es/blocks/product-details/) — ecommerce: página de detalle de producto con galería, variantes y agregar al carrito.
- [Portfolio Header](/es/blocks/portfolio-header/) — portafolio: encabezado de presentación.
- [Portfolio Sections](/es/blocks/portfolio-sections/) — portafolio: grid de showcase de trabajo.
- [Story](/es/blocks/story/) — images-ads: plantilla vertical 9:16.
