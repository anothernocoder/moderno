**Blog Sections** es un [block](/es/introduction/blocks/) de marketing: una sección de
listado/preview de posts con un encabezado centrado (kicker/título/subtítulo) sobre una grilla
responsiva de tarjetas de post, cada una con badge de categoría opcional, título, extracto y
autor. Compone los primitives `Card`, `Avatar` y `Badge` y los tokens `--md-*`. Lo copias a tu
repo con el CLI y lo editas libremente.

## Props

| Prop | Tipo | Default | Descripción |
|------|------|---------|-------------|
| `kicker` | `string` | `'Blog'` | Texto pequeño sobre el título |
| `title` | `string` | `'Últimas publicaciones'` | Título de la sección |
| `subtitle` | `string` | `'Notas del equipo sobre diseño, ingeniería y producto.'` | Texto de introducción corto |
| `posts` | `BlogPostPreview[]` | Tres posts de ejemplo | Las tarjetas de preview |

Un `BlogPostPreview` es `{ title: string; excerpt: string; author: string; date: string; avatarSrc?: string; initials?: string; category?: string }`.
Cada item se renderiza como un `Card`: un `Badge` opcional para `category` arriba, el título del
post como `<h3>`, el extracto como párrafo, y un footer con `Avatar` + nombre del autor + fecha.
Cuando falta `avatarSrc`, `Avatar` recurre a `initials`.

## Uso

```tsx
import { BlogSections } from './blocks/BlogSections'

<BlogSections
  kicker="Blog"
  title="Últimas publicaciones"
  subtitle="Notas del equipo sobre diseño, ingeniería y producto."
  posts={[
    {
      title: 'Cómo escalamos nuestro sistema de diseño a cuatro frameworks',
      excerpt: 'Componentes headless, una sola API, estilos compartidos.',
      author: 'Jamie Chen',
      date: '12 mar 2025',
      initials: 'JC',
      category: 'Ingeniería',
    },
  ]}
/>
```

La API de props es idéntica en los cuatro frameworks.

El block renderiza un encabezado centrado (kicker, título, subtítulo) sobre una grilla responsiva
de `Card`s construida a partir de `posts`, una tarjeta por post. La grilla usa
`repeat(auto-fit, minmax(280px, 1fr))`, así que las tarjetas se reacomodan de tres columnas a una
sola a medida que el viewport se angosta.
