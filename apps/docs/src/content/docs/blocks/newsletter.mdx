**Newsletter** is a marketing [block](/introduction/blocks/): an email signup section with a
centered kicker/title/subtitle header above an inline email form. It composes the `Input` and
`Button` primitives and the `--md-*` tokens. You copy it into your repo with the CLI and edit it
freely.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `kicker` | `string` | `'Newsletter'` | Small text above the title |
| `title` | `string` | `'Recibe las novedades de Moderno'` | Section title |
| `subtitle` | `string` | `'Un correo al mes con nuevos blocks, primitives y notas de diseño.'` | Short intro text |
| `placeholder` | `string` | `'tu@email.com'` | Email input placeholder |
| `buttonLabel` | `string` | `'Suscribirme'` | Submit button label |
| `defaultEmail` | `string` | `''` | Initial value for the email field |
| `onSubmit` | `(email: string) => void` | — | Called with the email address on submit (`preventDefault` is already handled) |

## Usage

```tsx
import { Newsletter } from './blocks/Newsletter'

<Newsletter
  kicker="Newsletter"
  title="Stay in the loop"
  subtitle="One email a month with new blocks, primitives, and design notes."
  placeholder="you@email.com"
  buttonLabel="Subscribe"
  onSubmit={(email) => console.log(email)}
/>
```

The props API is identical across all four frameworks (in Vue, `onSubmit` is the `@submit` event
instead of a prop).

The block renders a centered header (kicker, title, subtitle) above a single-row form with an
email `Input` and a submit `Button`. It manages its own field state internally and calls
`onSubmit` with the current email; wire that callback up to your own request logic.
