**Blog Header** es un [block](/es/introduction/blocks/) de marketing: un encabezado de
índice/landing para un blog, con un kicker/título/subtítulo centrados sobre una fila de chips de
categoría. Compone el primitive `Chip` y los tokens `--md-*`. Lo copias a tu repo con el CLI y lo
editas libremente.

## Props

| Prop | Tipo | Default | Descripción |
|------|------|---------|-------------|
| `kicker` | `string` | `'Blog'` | Texto pequeño sobre el título |
| `title` | `string` | `'Ideas, historias y aprendizajes del equipo'` | Título de la sección |
| `subtitle` | `string` | `'Reflexiones sobre diseño, ingeniería y producto…'` | Texto de introducción corto |
| `categories` | `string[]` | `['Producto', 'Ingeniería', 'Diseño']` | Chips de categoría debajo del subtítulo |

## Uso

```tsx
import { BlogHeader } from './blocks/BlogHeader'

<BlogHeader
  kicker="Blog"
  title="Ideas y aprendizajes del equipo"
  subtitle="Notas sobre diseño, ingeniería y producto, directo desde quienes construyen Moderno."
  categories={['Producto', 'Ingeniería', 'Diseño']}
/>
```

La API de props es idéntica en los cuatro frameworks.

El block renderiza un encabezado centrado (kicker, título, subtítulo) sobre una fila `flex-wrap`
de `Chip`s construida a partir de `categories`, un chip por categoría, con un pequeño margen
superior que la separa del subtítulo.
