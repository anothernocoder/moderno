**DatePicker** picks a single date or a date range from a calendar popover. A **compositional**
primitive (ADR-0003): `DatePicker.Root`, `DatePicker.Label`, `DatePicker.Control`,
`DatePicker.Input`, `DatePicker.Trigger`, `DatePicker.ClearTrigger`, `DatePicker.Content`.
The behavior (keyboard navigation, day/month/year views, range selection, popper
positioning, ARIA `grid`/`application`) comes from Zag's `date-picker` machine.
`DatePicker.Content` renders the full day/month/year calendar internally — unlike
`Select.Item`, its structure doesn't vary per consumer, so it isn't split into further
public parts.

## Selecting a range

Set `selectionMode="range"` to select a start and end date. The calendar renders two
selected endpoints and the days between them get `data-in-range`.

```tsx
<DatePicker.Root selectionMode="range">
  <DatePicker.Control>
    <DatePicker.Input index={0} />
    <DatePicker.Input index={1} />
    <DatePicker.Trigger />
  </DatePicker.Control>
  <DatePicker.Content />
</DatePicker.Root>
```

## Props (`DatePicker.Root`)

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `selectionMode` | `'single' \| 'range'` | `'single'` | Pick one date, or a start/end range. |
| `defaultValue` | `string[]` | — | Initially selected date(s), as ISO 8601 strings (uncontrolled). |
| `value` / `v-model` / `bind:value` | `string[]` | — | Selected date(s), as ISO 8601 strings (controlled). |
| `min` | `string` | — | Earliest selectable date (ISO 8601). |
| `max` | `string` | — | Latest selectable date (ISO 8601). |
| `disabled` | `boolean` | `false` | Disables the control. |
| `locale` | `string` | `'en-US'` | BCP 47 locale used to format and parse dates. |
| `startOfWeek` | `number` | — | `0` (Sunday) through `6` (Saturday). |
| `onValueChange` | `(value: string[]) => void` | — | Callback when the selection changes. |

`DatePicker.Input` accepts an optional `index` (`0` or `1`), used in range mode to render
separate start/end fields. `DatePicker.Trigger` and `DatePicker.ClearTrigger` accept custom
children in place of their default icon.
