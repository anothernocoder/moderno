**Cta** is a marketing [block](/introduction/blocks/): a focused call-to-action section — a title,
an optional short description, and one or two buttons. It's intentionally simpler than
[Hero](/blocks/hero/): no kicker eyebrow, no subtitle-as-secondary-emphasis treatment, just a plain
and punchy title, description, and button(s). It composes the `Button` primitive and the `--md-*`
tokens. You copy it into your repo with the CLI and edit it freely.

After copying it, install the `@moderno/<framework>` primitive and
[import the styles](/introduction/installation/). From there, the code is yours.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `title` | `string` | `'¿Listo para construir con Moderno?'` | Section title |
| `description` | `string` | One short sentence | Optional supporting copy |
| `ctaLabel` | `string` | `'Empezar ahora'` | Primary button label |
| `onCta` | `() => void` | — | Primary button click handler |
| `secondaryLabel` | `string` | `undefined` | Secondary button label; only renders when set |
| `onSecondary` | `() => void` | — | Secondary button click handler |

## Usage

```tsx
import { Cta } from './blocks/Cta'

<Cta
  title="Ready to build with Moderno?"
  description="Copy the block, install the primitive, and make it your own."
  ctaLabel="Get started"
  onCta={() => console.log('cta')}
  secondaryLabel="Talk to sales"
  onSecondary={() => console.log('secondary')}
/>
```

The props API is identical across all four frameworks.

The block renders a centered panel — title, optional description, and a row of one or two
`Button`s — on `var(--md-surface-raised)` with rounded corners, which visually sets it apart from
Hero's flat `--md-surface-base` band. Only the primary button renders by default; pass
`secondaryLabel` to show a second, secondary-variant button next to it.
