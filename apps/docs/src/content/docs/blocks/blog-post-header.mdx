**Blog Post Header** is a marketing [block](/introduction/blocks/): a single-article header with
an optional category badge, a large serif title, a subtitle, and a centered meta row with author
avatar, name, date, and read time. It composes the `Avatar` and `Badge` primitives and the
`--md-*` tokens. You copy it into your repo with the CLI and edit it freely.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `category` | `string` | `'Ingeniería'` | Category badge above the title |
| `title` | `string` | `'Cómo escalamos nuestro sistema de diseño a cuatro frameworks'` | Article title |
| `subtitle` | `string` | `'Una mirada al proceso detrás de construir Moderno…'` | Short intro text |
| `author` | `string` | `'Marta Gómez'` | Author name |
| `date` | `string` | `'12 mar 2025'` | Publish date |
| `readTime` | `string` | `'6 min de lectura'` | Estimated reading time |
| `avatarSrc` | `string` | — | Author avatar URL (uses initials if omitted) |
| `initials` | `string` | `'MG'` | Avatar fallback |

## Usage

```tsx
import { BlogPostHeader } from './blocks/BlogPostHeader'

<BlogPostHeader
  category="Engineering"
  title="How we scaled our design system to four frameworks"
  subtitle="A look at the process behind building Moderno."
  author="Jamie Chen"
  date="Mar 12, 2025"
  readTime="6 min read"
  initials="JC"
/>
```

The props API is identical across all four frameworks.

The block renders a centered `Badge` for `category` (if present) above a large serif `<h1>`
title and subtitle, followed by a centered meta row: `Avatar` + author name, then the date and
read time, each separated by a middot and only rendered when its data is present.
