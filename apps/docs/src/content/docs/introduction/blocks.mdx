Moderno ships through **two channels** (ADR-0001):

- **Primitives** → npm packages (`@moderno/react`, etc.). Versioned; fixes propagate from the
  center. They are the [components](/components/button/).
- **Blocks** → **copy-paste registry**. Layout compositions (Hero, Pricing, app shell)
  that you copy into your repo and edit freely. They don't update on their own — that's the
  trade-off for layout freedom.

## The CLI

```sh
# See the available blocks
npx @moderno/registry list

# Copy a block into your project, for your framework
npx @moderno/registry add hero --framework react --dest ./src/blocks
```

The command copies the block file (for the chosen framework) to `--dest`
(defaults to `./src/blocks`). From there the code is **yours**: edit it however you like.

After copying it, make sure you have the corresponding primitive package installed
(`@moderno/<framework>`) and that you've [imported the styles](/introduction/installation/).

## Available blocks

Each block has its own page with a live demo (all four frameworks), an install command, and a
props reference in the **Blocks** section of the menu:

- [Hero](/blocks/hero/) — marketing: above-the-fold hero.
- [Banner](/blocks/banner/) — marketing: dismissible top-of-page promo/announcement strip.
- [Pricing](/blocks/pricing/) — marketing: plan comparison section.
- [Header](/blocks/header/) — marketing: site header / navigation with logo, nav links, and CTA(s).
- [Features](/blocks/features/) — marketing: feature highlight grid with icon, title, and description.
- [Content](/blocks/content/) — marketing: general content/prose section with headings and body text.
- [Careers](/blocks/careers/) — marketing: jobs/openings section with role title, location, and apply CTA.
- [Footer](/blocks/footer/) — marketing: site footer with navigation columns, legal links, and social links.
- [Contact](/blocks/contact/) — marketing: contact section with contact info and a name/email/subject/message form.
- [FAQ](/blocks/faq/) — marketing: frequently-asked-questions section with a centered header above a Q&A list.
- [Testimonials](/blocks/testimonials/) — marketing: social-proof section with quote, author, and avatar cards.
- [Logos](/blocks/logos/) — marketing: trusted-by logo strip with an optional kicker line.
- [Team](/blocks/team/) — marketing: team section with member avatar, name, role, and bio cards.
- [CTA](/blocks/cta/) — marketing: focused headline + CTA button section, simpler than Hero.
- [Blog header](/blocks/blog-header/) — marketing: centered kicker/title/subtitle above a row of category chips for a blog index page.
- [Blog sections](/blocks/blog-sections/) — marketing: centered header above a responsive grid of post preview cards (title, excerpt, author, category).
- [Blog post header](/blocks/blog-post-header/) — marketing: single-article header with category badge, large title, subtitle, and an author/date/read-time meta row.
- [Newsletter](/blocks/newsletter/) — marketing: email signup section with a centered header above an inline email form.
- [Marketing Stats](/blocks/marketing-stats/) — marketing: metric/number highlight row for landing pages (big serif numbers, no cards).
- [Contact Modal](/blocks/contact-modal/) — marketing: modal-based contact/lead capture with a compact form and a Dialog-powered direct-contact trigger.
- [404](/blocks/not-found/) — marketing: 404 / error-state section with a big serif numeral, message, and CTA links back to the site.
- [Stats](/blocks/stats/) — application: row of KPIs.
- [Table](/blocks/table/) — application: data table with selection and bulk actions.
- [Feeds](/blocks/feeds/) — application: activity feed / timeline with avatar, actor/action, and timestamp.
- [Product Card](/blocks/product-card/) — ecommerce: product card.
- [Product Details](/blocks/product-details/) — ecommerce: product detail page with gallery, variants, and add-to-cart.
- [Portfolio Header](/blocks/portfolio-header/) — portfolio: introductory header.
- [Portfolio Sections](/blocks/portfolio-sections/) — portfolio: work showcase grid.
- [Story](/blocks/story/) — images-ads: vertical 9:16 template.
