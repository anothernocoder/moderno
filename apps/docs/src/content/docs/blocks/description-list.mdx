**Description List** is an application [block](/introduction/blocks/): a key/value detail
list for structured record data (an order or profile detail view). Each row shows a label
and a value — stacked on narrow layouts, side-by-side on wider ones. It composes the `Card`
primitive and the `--md-*` tokens.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `title` | `string` | `'Detalle del pedido'` | Heading shown in the `Card` title |
| `items` | `DescriptionItem[]` | 8 sample fields | Label/value pairs to render |

### `DescriptionItem`

| Field | Type | Description |
|-------|------|--------------|
| `label` | `string` | Field name |
| `value` | `string` | Pre-formatted value |

## Usage

```tsx
import { DescriptionList } from './blocks/DescriptionList'

<DescriptionList
  title="Order details"
  items={[
    { label: 'Order number', value: 'A-1104' },
    { label: 'Customer', value: 'Ada Corp' },
    { label: 'Status', value: 'Shipped' },
  ]}
/>
```

The props API is identical across all four frameworks.
