**Careers** is a marketing [block](/introduction/blocks/): a jobs/openings section for careers
pages — a list of open roles with title, location, and an apply CTA. It composes the `Badge`,
`Button`, and `Divider` primitives and the `--md-*` tokens. You copy it into your repo with the
CLI and edit it freely.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `kicker` | `string` | `'Carreras'` | Small text above the title |
| `title` | `string` | `'Construye el futuro de Moderno con nosotros'` | Main headline |
| `subtitle` | `string` | `'Buscamos personas curiosas para un equipo pequeño y remoto.'` | Supporting text |
| `roles` | `CareerRole[]` | Four sample roles | The open roles list |

A `CareerRole` is `{ title: string; location: string; department?: string; type?: string; applyLabel?: string }`.

React, Svelte, and Solid also accept an `onApply?: (role: CareerRole) => void` callback fired when
a row's apply button is clicked. Vue emits an `apply` event with the same payload instead
(`@apply="..."`), following Vue's event-based convention.

## Usage

```tsx
import { Careers } from './blocks/Careers'

<Careers
  kicker="Careers"
  title="Come build with us"
  subtitle="We're a small, remote-first team hiring across the stack."
  roles={[
    { title: 'Senior Product Engineer', location: 'Remote (US)', department: 'Engineering', type: 'Full-time' },
    { title: 'Product Designer', location: 'Remote', department: 'Design', type: 'Full-time' },
  ]}
  onApply={(role) => console.log('apply', role.title)}
/>
```

The props API is identical across all four frameworks (React/Svelte/Solid use `onApply`; Vue uses
the `apply` emit).

Roles render as rows in a single bordered list (max-width `760px`), each with a title, a row of
badges for location/department/type, and an apply button on the right. A `Divider` separates rows
after the first one. Omit `department` or `type` per role to show fewer badges.
