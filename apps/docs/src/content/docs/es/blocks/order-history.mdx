**Order History** es un [block](/es/introduction/blocks/) de ecommerce: una lista de los
pedidos anteriores de un cliente — cada fila muestra el número de pedido, la fecha, una
píldora de estado y el total — que enlaza al detalle del pedido. Compone los primitives
`Card` y `Badge` junto con los tokens `--md-*`. Lo copias a tu repo con el CLI y lo
editas libremente.

Al igual que [Order Summaries](/es/blocks/order-summaries/), este block es puramente
presentacional: el `href` de cada pedido es un enlace simple, sin lógica de enrutamiento
— conéctalo al router de tu framework o déjalo como un enlace normal a una página de
detalle renderizada en el servidor.

## Props

| Prop | Tipo | Default | Descripción |
|------|------|---------|-------------|
| `heading` | `string` | `'Historial de pedidos'` | Encabezado sobre la lista (omítelo para ocultarlo) |
| `orders` | `OrderHistoryOrder[]` | 4 pedidos de ejemplo | Pedidos anteriores — cada uno `{ id, orderNumber, date, status, total, href }` |
| `emptyMessage` | `string` | `'Todavía no tienes pedidos.'` | Mensaje mostrado cuando `orders` está vacío |
| `detailsLabel` | `string` | `'Ver detalles'` | Texto del enlace de detalle de cada pedido |

`status` es uno de `'delivered' | 'shipping' | 'processing' | 'cancelled'`, mapeado
internamente a una etiqueta en español y una variante de `Badge`:

| Estado | Etiqueta | Variante de Badge |
|--------|----------|--------------------|
| `delivered` | Entregado | `success` |
| `shipping` | En camino | `info` |
| `processing` | Procesando | `warning` |
| `cancelled` | Cancelado | `error` |

## Uso

```tsx
import { OrderHistory } from './blocks/OrderHistory'

<OrderHistory
  orders={[
    { id: 'ord-1', orderNumber: '1042', date: '15 de julio, 2026', status: 'delivered', total: 458000, href: '/pedidos/1042' },
    { id: 'ord-2', orderNumber: '1039', date: '2 de julio, 2026', status: 'shipping', total: 210000, href: '/pedidos/1039' },
  ]}
/>
```

La API de props es idéntica en los cuatro frameworks. El `total` de cada pedido es un
`number` simple (formateado internamente con `toLocaleString`), y `href` se renderiza
como una etiqueta `<a>` simple — no incluye lógica de enrutamiento del lado del cliente,
así que conecta el componente `Link` del router de tu framework si necesitas uno. Sin
primitives nuevos, sin lógica de interacción personalizada — solo los primitives
`Card` y `Badge` existentes compuestos con layout.
