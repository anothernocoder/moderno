**Careers** es un [block](/es/introduction/blocks/) de marketing: una sección de
carreras/vacantes para páginas de empleo — una lista de roles abiertos con título, ubicación y
un CTA de aplicar. Compone los primitives `Badge`, `Button` y `Divider` y los tokens `--md-*`.
Lo copias a tu repo con el CLI y lo editas libremente.

## Props

| Prop | Tipo | Default | Descripción |
|------|------|---------|-------------|
| `kicker` | `string` | `'Carreras'` | Texto pequeño sobre el título |
| `title` | `string` | `'Construye el futuro de Moderno con nosotros'` | Titular principal |
| `subtitle` | `string` | `'Buscamos personas curiosas para un equipo pequeño y remoto.'` | Texto de apoyo |
| `roles` | `CareerRole[]` | Cuatro roles de ejemplo | La lista de roles abiertos |

Un `CareerRole` es `{ title: string; location: string; department?: string; type?: string; applyLabel?: string }`.

React, Svelte y Solid también aceptan un callback `onApply?: (role: CareerRole) => void` que se
dispara al hacer clic en el botón de aplicar de una fila. Vue en cambio emite un evento `apply`
con el mismo payload (`@apply="..."`), siguiendo la convención de eventos de Vue.

## Uso

```tsx
import { Careers } from './blocks/Careers'

<Careers
  kicker="Carreras"
  title="Ven a construir con nosotros"
  subtitle="Somos un equipo pequeño y remoto contratando en todo el stack."
  roles={[
    { title: 'Ingeniera/o de Producto Senior', location: 'Remoto', department: 'Ingeniería', type: 'Tiempo completo' },
    { title: 'Diseñadora/or de Producto', location: 'Remoto', department: 'Diseño', type: 'Tiempo completo' },
  ]}
  onApply={(role) => console.log('aplicar', role.title)}
/>
```

La API de props es idéntica en los cuatro frameworks (React/Svelte/Solid usan `onApply`; Vue usa
el emit `apply`).

Los roles se renderizan como filas en una sola lista bordeada (max-width `760px`), cada una con
un título, una fila de badges para ubicación/departamento/tipo, y un botón de aplicar a la
derecha. Un `Divider` separa las filas a partir de la segunda. Omite `department` o `type` por
rol para mostrar menos badges.
