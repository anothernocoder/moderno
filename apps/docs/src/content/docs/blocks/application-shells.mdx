**Application Shells** is an application [block](/introduction/blocks/): the outer frame most
other application blocks (like [Table](/blocks/table/) or [Stats](/blocks/stats/)) live inside.
It composes a collapsible sidebar (brand, nav links with an active-item indicator, and a
user/sign-out footer), a topbar (sidebar toggle, page title, notifications, and a user menu), and
a scrollable main content region. It composes the `Avatar`, `Badge`, `Button`, and `Divider`
primitives and the `--md-*` tokens. You copy it into your repo with the CLI and edit it freely.

The demo below is height-constrained to fit this page — mount the real block inside a full-height
container (e.g. `#root { height: 100vh }`) and it fills 100% of its parent.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `brandLabel` | `string` | `'Moderno'` | Brand/logo text shown at the top of the sidebar |
| `brandHref` | `string` | `'#'` | `href` for the brand link |
| `navItems` | `ApplicationShellsNavItem[]` | 5 sample items | Sidebar navigation links |
| `defaultActiveItem` | `string` | first item's `id` | Initially active nav item id |
| `pageTitle` | `string` | active nav item's label | Topbar heading; overrides the derived title |
| `user` | `ApplicationShellsUser` | sample user | Rendered in the sidebar footer and topbar user menu |
| `unreadCount` | `number` | `3` | Badge count next to the notifications button; hidden when `0` |
| `onNavItemClick` | `(item: ApplicationShellsNavItem) => void` | — | Called when a nav item is clicked, with the clicked item |
| `onSignOut` | `() => void` | — | Called when "Sign out" is clicked |
| `onNotificationsClick` | `() => void` | — | Called when the notifications button is clicked |
| `onUserMenuClick` | `() => void` | — | Called when the topbar user button is clicked |

The main content region renders its children/slot; when none are passed, it falls back to a
placeholder that shows the active nav item's label.

`ApplicationShellsNavItem` shape:

| Field | Type | Description |
|-------|------|--------------|
| `id` | `string` | Unique id, also used as the active/selection key |
| `label` | `string` | Link text |
| `href` | `string` | Optional `href` (defaults to `#`) |

`ApplicationShellsUser` shape:

| Field | Type | Description |
|-------|------|--------------|
| `name` | `string` | Display name |
| `role` | `string` | Optional role/subtitle shown under the name in the sidebar footer |
| `avatarInitials` | `string` | Fallback initials for the `Avatar` |
| `avatarSrc` | `string` | Optional avatar image URL |

## Usage

```tsx
import { ApplicationShells } from './blocks/ApplicationShells'

<ApplicationShells
  navItems={[
    { id: 'overview', label: 'Overview', href: '/app' },
    { id: 'orders', label: 'Orders', href: '/app/orders' },
    { id: 'settings', label: 'Settings', href: '/app/settings' },
  ]}
  user={{ name: 'Jane Doe', role: 'Admin', avatarInitials: 'JD' }}
  onNavItemClick={(item) => router.push(item.href)}
  onSignOut={() => signOut()}
>
  <YourPageContent />
</ApplicationShells>
```

The props API is identical across all four frameworks. In Vue, `onNavItemClick`/`onSignOut`/
`onNotificationsClick`/`onUserMenuClick` are the `@nav-item-click`/`@sign-out`/
`@notifications-click`/`@user-menu-click` events; in the others they're callback props. Content
goes through the default slot/`children` prop in every framework.

The sidebar toggle and the nav active-item indicator are local component state — there's no
router or new primitive involved. Wire `onNavItemClick` (and real `href`s) up to your router of
choice to drive real navigation.
