**Category Preview** is an ecommerce [block](/introduction/blocks/): a grid of
category tiles — an image and a label each — linking into a storefront's
category pages (e.g. "Chairs", "Tables", "Lighting"). Plain layout composed
with `--md-*` tokens — no primitives, no bespoke interaction logic. You copy
it into your repo with the CLI and edit it freely.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `heading` | `string` | `'Comprar por categoría'` | Heading above the grid (omit to hide it) |
| `categories` | `CategoryPreviewItem[]` | 4 sample categories | Tiles to render — each with `id`, `label`, `image`, and `href` |
| `onCategoryClick` | `(category: CategoryPreviewItem) => void` | — | Called when a tile is clicked |

## Usage

```tsx
import { CategoryPreview } from './blocks/CategoryPreview'

<CategoryPreview
  heading="Shop by category"
  categories={[
    { id: 'chairs', label: 'Chairs', image: '/img/chairs.jpg', href: '/categories/chairs' },
    { id: 'tables', label: 'Tables', image: '/img/tables.jpg', href: '/categories/tables' },
  ]}
  onCategoryClick={(category) => trackCategoryClick(category.id)}
/>
```

The props API is identical across all four frameworks. In Vue, `onCategoryClick`
is the `@category-click` event; in React, Svelte, and Solid it's a callback
prop. Each tile is just an anchor wrapping an image and a label — no new
primitive is introduced.
