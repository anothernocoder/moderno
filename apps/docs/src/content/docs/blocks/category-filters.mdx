**Category Filters** is an ecommerce [block](/introduction/blocks/): faceted sidebar
filters for a product listing page — a searchable category combobox, a size toggle
group, and a min/max price range, with "Apply" and "Reset" actions. It composes the
`Combobox`, `Button`, and `NumberInput` primitives and the `--md-*` tokens. You copy
it into your repo with the CLI and edit it freely.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `heading` | `string` | `'Filtros'` | Heading above the filters (omit to hide it) |
| `categoryLabel` | `string` | `'Categoría'` | Label for the category `Combobox` |
| `categoryPlaceholder` | `string` | `'Buscar categoría'` | Placeholder text for the category input |
| `categories` | `ComboboxItem[]` | 5 sample categories | Options for the category `Combobox` — each with `label` and `value` |
| `defaultCategory` | `string` | — | Initially selected category value |
| `sizeLabel` | `string` | `'Talla'` | Label above the size toggle group |
| `sizes` | `string[]` | `['S', 'M', 'L', 'XL']` | Selectable size options |
| `defaultSize` | `string` | — | Initially selected size |
| `priceLabel` | `string` | `'Precio'` | Label above the price range |
| `minPriceLabel` | `string` | `'Mínimo'` | Label for the minimum price `NumberInput` |
| `maxPriceLabel` | `string` | `'Máximo'` | Label for the maximum price `NumberInput` |
| `defaultMinPrice` | `number` | — | Initial minimum price |
| `defaultMaxPrice` | `number` | — | Initial maximum price |
| `applyLabel` | `string` | `'Aplicar filtros'` | Text for the apply button |
| `resetLabel` | `string` | `'Limpiar'` | Text for the reset button |
| `onApply` | `(filters: { category?: string; size?: string; minPrice?: number; maxPrice?: number }) => void` | — | Called when "Apply" is clicked, with the current filter state |
| `onReset` | `() => void` | — | Called when "Reset" is clicked, after the internal state is cleared |

## Usage

```tsx
import { CategoryFilters } from './blocks/CategoryFilters'

<CategoryFilters
  categories={[
    { label: 'Chairs', value: 'chairs' },
    { label: 'Tables', value: 'tables' },
    { label: 'Lighting', value: 'lighting' },
  ]}
  sizes={['S', 'M', 'L', 'XL']}
  onApply={(filters) => applyProductFilters(filters)}
  onReset={() => clearProductFilters()}
/>
```

The props API is identical across all four frameworks. In Vue, `onApply` and `onReset`
are the `@apply` and `@reset` events; in the others they're callback props. The
category facet is a searchable `Combobox` — typing filters the list by label, and
selection, keyboard interaction, and ARIA all come from the primitive; size is a
simple toggle-button group, and price is a min/max pair of `NumberInput`s — no new
primitives, no bespoke interaction logic.
