**Portfolio Sections** is a portfolio [block](/introduction/blocks/): a work showcase
grid of project cards with an image, title, category/date meta, and a hover arrow
link to the case study. It's built with the `--md-*` tokens only, no primitives.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `projects` | `PortfolioProject[]` | 4 sample projects | Projects to render in the grid |
| `onSelect` | `(project: PortfolioProject) => void` | — | Called when a card is clicked |

### `PortfolioProject`

| Field | Type | Description |
|-------|------|-------------|
| `title` | `string` | Project title |
| `meta` | `string` | Category / date meta line |
| `image` | `string` | Cover image URL (placeholder if omitted) |
| `href` | `string` | Case study destination URL |

## Usage

```tsx
import { PortfolioSections } from './blocks/PortfolioSections'

<PortfolioSections
  projects={[
    { title: 'Moderno design system', meta: 'Design systems · 2025', href: '/work/moderno' },
    { title: 'Rappi Turbo checkout', meta: 'Product · 2024', href: '/work/rappi-turbo' },
  ]}
  onSelect={(project) => track('portfolio_card_click', project)}
/>
```

The props API is identical across all four frameworks. In Vue, `onSelect` is the
`@select` event; in the others it's a callback prop.
