**List** es un [block](/es/introduction/blocks/) de aplicación: compañero de [Table](/es/blocks/table/)
para contextos donde una lista simple basada en filas se lee mejor que una grilla de datos. Cada
fila muestra un avatar, texto primario/secundario, un badge de estado opcional, texto meta
opcional y una acción de fila — sin ordenamiento, sin selección múltiple y sin columnas
tabulares. Compone los primitives `Avatar`, `Badge`, `Button` y `Divider`, y los tokens `--md-*`.

## Props

| Prop | Tipo | Default | Descripción |
|------|------|---------|-------------|
| `items` | `ListItem[]` | 4 registros de muestra | Las filas a renderizar |
| `onItemAction` | `(item: ListItem) => void` | — | Se llama al hacer clic en el botón `⋯` de una fila |

Forma de `ListItem`:

| Campo | Tipo | Descripción |
|-------|------|--------------|
| `id` | `string` | Id único de la fila |
| `title` | `string` | Texto primario |
| `subtitle` | `string` | Texto secundario (omitido si no se define) |
| `meta` | `string` | Texto meta al final, p. ej. una fecha/hora (oculto si se omite) |
| `avatarSrc` | `string` | URL de la imagen del avatar |
| `avatarInitials` | `string` | Iniciales de respaldo del avatar, mostradas cuando falta `avatarSrc` |
| `status` | `string` | Etiqueta de estado mostrada en el `Badge` (oculta si se omite) |
| `statusVariant` | `'neutral' \| 'solid' \| 'success' \| 'warning' \| 'error' \| 'info'` | Variante de color del `Badge` para el estado |

## Uso

```tsx
import { List } from './blocks/List'

<List
  items={[
    { id: '1', title: 'Acme Co', subtitle: 'billing@acme.co', avatarInitials: 'AC', status: 'Activo', statusVariant: 'success', meta: 'Hace 2 h' },
    { id: '2', title: 'Globex', subtitle: 'ops@globex.com', avatarInitials: 'GX', status: 'Pendiente', statusVariant: 'warning', meta: 'Hace 1 d' },
  ]}
  onItemAction={(item) => abrirMenuDeItem(item)}
/>
```

La API de props es idéntica en los cuatro frameworks. En Vue, `onItemAction` es el evento
`@item-action`; en los demás es una prop callback.
