**Alerts** is an application [block](/introduction/blocks/): a stacked group of inline
info/success/warning/error messages composed into a realistic dashboard/form alerts area,
with dismiss and inline-action affordances. It composes the `Alert` and `Button`
[primitives](/components/alert/) and the `--md-*` tokens — it does not reimplement the
alert itself, only the surrounding layout and interaction.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `items` | `AlertItem[]` | 4 sample alerts (info/success/warning/error) | Messages to render |

### `AlertItem`

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `id` | `string` | — | Stable key, used to track dismissal |
| `variant` | `'info' \| 'success' \| 'warning' \| 'error'` | — | Passed straight through to `Alert` |
| `title` | `string` | — | Optional bold title |
| `description` | `string` | — | Message body |
| `dismissible` | `boolean` | `false` | Shows a close button that removes the alert from local state |
| `actionLabel` | `string` | — | Shows an inline `Button` (e.g. "Retry payment") next to the message |

## Usage

```tsx
import { Alerts } from './blocks/Alerts'

<Alerts
  items={[
    { id: 'saved', variant: 'success', title: 'Saved', description: 'Your changes are live.', dismissible: true },
    { id: 'failed', variant: 'error', title: 'Payment failed', description: 'Check your card details.', actionLabel: 'Retry payment' },
  ]}
/>
```

The props API is identical across all four frameworks.
