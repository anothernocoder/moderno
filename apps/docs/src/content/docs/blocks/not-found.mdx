**NotFound** is a marketing [block](/introduction/blocks/): a 404 / error-state section with a
huge serif numeral above a title, a description, and CTA link(s) back to the site. There's no
illustration asset — it's a purely typographic treatment, consistent with how other blocks in this
repo avoid binary image assets. It composes the `--md-*` tokens only. You copy it into your repo
with the CLI and edit it freely.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `code` | `string` | `'404'` | The big serif numeral |
| `title` | `string` | `'Página no encontrada'` | Heading below the numeral |
| `description` | `string` | `'La página que buscas no existe o fue movida.'` | Supporting text |
| `homeLabel` | `string` | `'Volver al inicio'` | Label for the primary link back to the site |
| `homeHref` | `string` | `'/'` | `href` for the primary link |
| `secondaryLabel` | `string` | `undefined` | Optional label for a second, outline-styled link |
| `secondaryHref` | `string` | `undefined` | Optional `href` for the secondary link |

`homeLabel`/`homeHref` and `secondaryLabel`/`secondaryHref` render as real `<a>` elements (not the
`Button` primitive), since they're navigation, not callbacks. The secondary link only renders when
both `secondaryLabel` and `secondaryHref` are provided.

## Usage

```tsx
import { NotFound } from './blocks/NotFound'

<NotFound
  code="404"
  title="Page not found"
  description="The page you're looking for doesn't exist or was moved."
  homeLabel="Back to home"
  homeHref="/"
  secondaryLabel="Contact support"
  secondaryHref="/contact"
/>
```

The props API is identical across all four frameworks.

The block renders a centered section with generous vertical padding: the `code` numeral in a huge
serif typeface, `title` as a smaller heading, `description` as body copy, and a centered row of
link CTAs — `homeHref` styled to look like a primary button, and, when provided,
`secondaryHref` styled as a secondary/outline link next to it.
