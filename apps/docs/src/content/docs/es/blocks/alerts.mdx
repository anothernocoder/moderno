**Alerts** es un [block](/es/introduction/blocks/) de aplicación: un grupo apilado de
mensajes info/success/warning/error compuesto en un área de alertas realista de dashboard/
formulario, con cierre y acciones en línea. Compone los primitives `Alert` y `Button`
([ver](/es/components/alert/)) y los tokens `--md-*` — no reimplementa la alerta en sí,
solo el layout y la interacción que la rodean.

## Props

| Prop | Tipo | Default | Descripción |
|------|------|---------|-------------|
| `items` | `AlertItem[]` | 4 alertas de muestra (info/success/warning/error) | Mensajes a renderizar |

### `AlertItem`

| Campo | Tipo | Default | Descripción |
|-------|------|---------|-------------|
| `id` | `string` | — | Clave estable, usada para rastrear el cierre |
| `variant` | `'info' \| 'success' \| 'warning' \| 'error'` | — | Se pasa directo a `Alert` |
| `title` | `string` | — | Título en negrita opcional |
| `description` | `string` | — | Cuerpo del mensaje |
| `dismissible` | `boolean` | `false` | Muestra un botón de cierre que quita la alerta del estado local |
| `actionLabel` | `string` | — | Muestra un `Button` en línea (ej. "Reintentar pago") junto al mensaje |

## Uso

```tsx
import { Alerts } from './blocks/Alerts'

<Alerts
  items={[
    { id: 'saved', variant: 'success', title: 'Guardado', description: 'Tus cambios ya están en línea.', dismissible: true },
    { id: 'failed', variant: 'error', title: 'Pago fallido', description: 'Revisa los datos de tu tarjeta.', actionLabel: 'Reintentar pago' },
  ]}
/>
```

La API de props es idéntica en los cuatro frameworks.
