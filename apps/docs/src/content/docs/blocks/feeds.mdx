**Feeds** is an application [block](/introduction/blocks/): a vertical activity-feed
timeline. Each entry shows an avatar connected to the next by a rail line, an
actor + action sentence, an optional status tag, and a timestamp. It composes the
`Avatar` and `Badge` primitives and the `--md-*` tokens.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `items` | `FeedItem[]` | 4 sample entries | Activity entries to render |

### `FeedItem`

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `id` | `string` | — | Unique key |
| `actor` | `string` | — | Who performed the action |
| `initials` | `string` | — | Fallback initials shown in the `Avatar` |
| `action` | `string` | — | Action sentence, rendered after the actor's name |
| `timestamp` | `string` | — | Pre-formatted relative/absolute time |
| `tag` | `string` | — | Optional status label (hidden if omitted) |
| `tagVariant` | `'neutral' \| 'solid' \| 'success' \| 'warning' \| 'error' \| 'info'` | — | `Badge` color for `tag` |

## Usage

```tsx
import { Feeds } from './blocks/Feeds'

<Feeds
  items={[
    { id: '1', actor: 'Ada Lovelace', initials: 'AL', action: 'merged a pull request', tag: 'Merged', tagVariant: 'success', timestamp: '5m ago' },
    { id: '2', actor: 'Grace Hopper', initials: 'GH', action: 'commented on an issue', timestamp: '32m ago' },
  ]}
/>
```

The props API is identical across all four frameworks.
