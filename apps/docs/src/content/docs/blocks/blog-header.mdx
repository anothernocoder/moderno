**Blog Header** is a marketing [block](/introduction/blocks/): an index/landing header for a blog,
with a centered kicker/title/subtitle above a row of category chips. It composes the `Chip`
primitive and the `--md-*` tokens. You copy it into your repo with the CLI and edit it freely.

[import the styles](/introduction/installation/). From there, the code is yours.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `kicker` | `string` | `'Blog'` | Small text above the title |
| `title` | `string` | `'Ideas, historias y aprendizajes del equipo'` | Section title |
| `subtitle` | `string` | `'Reflexiones sobre diseño, ingeniería y producto…'` | Short intro text |
| `categories` | `string[]` | `['Producto', 'Ingeniería', 'Diseño']` | Category chips shown below the subtitle |

## Usage

```tsx
import { BlogHeader } from './blocks/BlogHeader'

<BlogHeader
  kicker="Blog"
  title="Ideas and lessons from the team"
  subtitle="Notes on design, engineering, and product, straight from the people building Moderno."
  categories={['Product', 'Engineering', 'Design']}
/>
```

The props API is identical across all four frameworks.

The block renders a centered header (kicker, title, subtitle) above a `flex-wrap` row of `Chip`s
built from `categories`, one chip per category, with a small top margin separating it from the
subtitle.
