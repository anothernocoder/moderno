**Contenedores** es un [block](/es/introduction/blocks/) de aplicaciones: un conjunto de
envoltorios de layout de página con nombre — angosto, predeterminado, ancho y completo — cada
uno combinando un `max-width` con padding horizontal consistente para centrar el contenido de
la página a un ancho determinado. Compone los primitives `Card` y `Badge` y los tokens
`--md-*` — sin lógica de interacción a medida, solo una referencia de layout copy-paste de la
que puedes elegir una variante.

## Props

| Prop | Tipo | Default | Descripción |
|------|------|---------|-------------|
| `title` | `string` | `'Contenedores'` | Encabezado del panel |
| `description` | `string` | Texto de ejemplo | Subtítulo del panel (se oculta si está vacío) |
| `variants` | `ContainerVariant[]` | 4 variantes de ejemplo | Los anchos de contenedor a demostrar |

Forma de `ContainerVariant`:

| Campo | Tipo | Descripción |
|-------|------|--------------|
| `id` | `string` | Id único de la variante / key de lista |
| `name` | `string` | Etiqueta de la variante (p. ej. "Angosto", "Ancho") |
| `maxWidth` | `string` | Valor CSS `max-width` aplicado al envoltorio, p. ej. `'640px'` o `'none'` para borde a borde |
| `description` | `string` | Cuándo usar esta variante |

## Uso

```tsx
import { Containers } from './blocks/Containers'

<Containers
  title="Anchos de página"
  variants={[
    { id: 'narrow', name: 'Angosto', maxWidth: '640px', description: 'Contenido de lectura enfocado, formularios, páginas de ajustes.' },
    { id: 'wide', name: 'Ancho', maxWidth: '1024px', description: 'Vistas densas en datos: tablas, dashboards de varias columnas.' },
  ]}
/>
```

La API de props es idéntica en los cuatro frameworks. Cada variante se renderiza dentro de un
envoltorio "viewport" con borde punteado (`.md-containers__viewport` en Vue/Svelte, el objeto
`viewportStyle` en React/Solid) para que puedas ver el contenedor de `max-width` centrado
contra el ancho disponible. El envoltorio de layout real es el elemento interno — `margin: 0
auto` más `padding: 0 24px` — que es la parte que copias a tu propio shell de página. No hay
cambio de breakpoints responsivo ni otra lógica de interacción por encima; agrégala una vez que
lo hayas copiado a tu repo.
