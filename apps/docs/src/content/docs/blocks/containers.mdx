**Containers** is an application [block](/introduction/blocks/): a set of named page layout
wrappers — narrow, default, wide, and full-bleed — each pairing a `max-width` with consistent
horizontal padding to center page content at a given width. It composes the `Card` and `Badge`
primitives and the `--md-*` tokens — no bespoke interaction logic, just a copy-paste layout
reference you can pick a variant from.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `title` | `string` | `'Contenedores'` | Panel heading |
| `description` | `string` | Sample copy | Panel subheading (hidden if empty) |
| `variants` | `ContainerVariant[]` | 4 sample variants | The container widths to demonstrate |

`ContainerVariant` shape:

| Field | Type | Description |
|-------|------|--------------|
| `id` | `string` | Unique variant id / list key |
| `name` | `string` | Variant label (e.g. "Narrow", "Wide") |
| `maxWidth` | `string` | CSS `max-width` value applied to the wrapper, e.g. `'640px'` or `'none'` for full-bleed |
| `description` | `string` | When to reach for this variant |

## Usage

```tsx
import { Containers } from './blocks/Containers'

<Containers
  title="Page widths"
  variants={[
    { id: 'narrow', name: 'Narrow', maxWidth: '640px', description: 'Focused reading content, forms, settings pages.' },
    { id: 'wide', name: 'Wide', maxWidth: '1024px', description: 'Data-dense views: tables, multi-column dashboards.' },
  ]}
/>
```

The props API is identical across all four frameworks. Each variant renders inside a dashed
"viewport" wrapper (`.md-containers__viewport` in Vue/Svelte, the `viewportStyle` object in
React/Solid) so you can see the centered `max-width` container against the available width. The
actual layout wrapper is the inner element — `margin: 0 auto` plus `padding: 0 24px` — which is
the part you copy into your own page shell. There's no responsive breakpoint switching or other
interaction logic layered on top; add that once you've copied it into your repo.
