**Shopping Cart** is an ecommerce [block](/introduction/blocks/): a cart drawer/page
listing line items — each with a thumbnail, title, unit price, a quantity stepper, and a
remove action — followed by a subtotal and a checkout CTA. It composes the `Card`,
`Button`, and `NumberInput` primitives plus the `--md-*` tokens. You copy it into your
repo with the CLI and edit it freely.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `heading` | `string` | `'Carrito de compras'` | Heading above the section (omit to hide it) |
| `items` | `ShoppingCartItem[]` | 3 sample line items | Cart line items — each `{ id, title, price, image?, quantity?, maxQuantity? }`; copied into local state on mount |
| `subtotalLabel` | `string` | `'Subtotal'` | Label for the subtotal row |
| `checkoutLabel` | `string` | `'Finalizar compra'` | Checkout CTA button text |
| `removeLabel` | `string` | `'Quitar'` | Remove-line-item button text |
| `emptyMessage` | `string` | `'Tu carrito está vacío.'` | Message shown when the cart has no items left |
| `onCheckout` | `(payload: { items: ShoppingCartItem[]; subtotal: number }) => void` | — | Called when the checkout CTA is clicked, with the current line items and subtotal |

## Usage

```tsx
import { ShoppingCart } from './blocks/ShoppingCart'

<ShoppingCart
  items={[
    { id: 'chair', title: 'Oak Dining Chair', price: 389000, quantity: 1 },
    { id: 'lamp', title: 'Arc Floor Lamp', price: 128000, quantity: 2 },
  ]}
  onCheckout={({ items, subtotal }) => checkout.start({ items, subtotal })}
/>
```

The props API is identical across all four frameworks. In Vue, `onCheckout` is the
`@checkout` event; in the others it's a callback prop. Each line item's `price` is a
plain `number` (formatted internally with `toLocaleString`) so the block can compute a
per-item line total and the running subtotal as quantities change. Quantity is tracked as
local component state per item, seeded from the `items` prop on mount and updated through
the `NumberInput` primitive's stepper — the same pattern used by the
[Product Details](/blocks/product-details/) block's quantity field. Removing a line item
filters it out of that local state. No new primitives, no bespoke stepper — just the
existing `Card`, `Button`, and `NumberInput` composed together.
