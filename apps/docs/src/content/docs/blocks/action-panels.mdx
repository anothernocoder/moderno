**Action panels** is an application [block](/introduction/blocks/): a list of settings-style
rows, each pairing a title and description on the left with a single action on the right —
the classic "Danger Zone" pattern for account/settings screens. Any row can opt into a
destructive style (for a delete-account action, for example). It composes the `Card`,
`Button`, and `Divider` primitives and the `--md-*` tokens.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `items` | `ActionPanelItem[]` | 3 sample panels | The rows to render |
| `onAction` | `(item: ActionPanelItem) => void` | — | Called with the row's data when its action button is clicked |

`ActionPanelItem` shape:

| Field | Type | Description |
|-------|------|--------------|
| `title` | `string` | Row heading |
| `description` | `string` | Supporting copy shown under the title |
| `actionLabel` | `string` | Label for the row's single action button |
| `destructive` | `boolean` | Colors the title and action with the `--md-error` token (danger-zone style) |

## Usage

```tsx
import { ActionPanels } from './blocks/ActionPanels'

<ActionPanels
  items={[
    { title: 'Download your data', description: 'Export everything as a JSON file.', actionLabel: 'Export' },
    {
      title: 'Delete account',
      description: 'This is permanent and cannot be undone.',
      actionLabel: 'Delete account',
      destructive: true,
    },
  ]}
  onAction={(item) => handlePanelAction(item)}
/>
```

The props API is identical across all four frameworks. In Vue, `onAction` is the `@action`
event instead of a callback prop.
