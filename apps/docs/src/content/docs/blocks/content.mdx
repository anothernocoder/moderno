**Content** is a marketing [block](/introduction/blocks/): a general content/prose section for
long-form marketing copy, broken into headed sections with paragraphs. It composes the
`Divider` primitive and the `--md-*` tokens. You copy it into your repo with the CLI and edit
it freely.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `kicker` | `string` | `'Contenido'` | Small text above the title |
| `title` | `string` | `'Escrito para durar, no solo para lanzar'` | Main headline |
| `subtitle` | `string` | `'Secciones de contenido largo con jerarquía tipográfica clara, listas para copiar en tu repo.'` | Supporting text |
| `sections` | `{ heading?: string; body: string[] }[]` | Three sample sections | The prose sections. `heading` is optional; `body` is a list of paragraphs |

## Usage

```tsx
import { Content } from './blocks/Content'

<Content
  kicker="About"
  title="Why we built this"
  subtitle="A short read on the problem and the approach."
  sections={[
    {
      heading: 'The problem',
      body: [
        'One paragraph of body copy.',
        'A second paragraph, if the section needs more than one.',
      ],
    },
    {
      heading: 'The approach',
      body: ['A single paragraph is fine too.'],
    },
  ]}
/>
```

The props API is identical across all four frameworks.

Sections render in a single centered column (max-width `720px`), each with an optional `h3`
heading followed by its paragraphs. A `Divider` separates sections after the first one. Omit
`heading` per section for plain body copy with no sub-heading.
